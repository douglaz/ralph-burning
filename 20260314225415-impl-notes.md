# Implementation Notes

## Decisions Made
- Reworked `ProcessBackendAdapter::cancel()` to remove the child from `active_children` before waiting, honor only a 500ms grace period after `SIGTERM`, and hand off stubborn children to a detached `SIGKILL` + reap task so the adapter never blocks the authoritative timeout/cancellation path.
- Reused `history_record_base_id(..., rollback_count = 0)` for workflow-stage `invocation_id` generation so invocation IDs, payload/artifact history IDs, and Codex temp-file names all share the same completion-round-aware format.
- Added regression coverage for the two remaining Round 3 issues: a `SIGTERM`-ignoring fake backend proving bounded cancellation, a `kill_on_drop(true)` safety-net test, and a workflow test that records planning invocation IDs across a completion-round restart.

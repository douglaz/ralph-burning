# Implementation Notes

## Decisions Made
- Added durable rollback checkpoints as first-class persisted records under `projects/<project-id>/rollback/`, and emitted matching `rollback_created` journal events so visible rollback history can be reconstructed from the append-only journal.
- Implemented rollback history filtering as journal replay logic that prunes abandoned branches on each `rollback_performed` event, which lets `run history`, `run tail`, and `run resume` hide rolled-back work while still showing new work after a rollback.
- Captured rollback checkpoints only at durable stage-transition boundaries after snapshot/journal transition side effects completed, so restoring a checkpoint yields a resumable state the engine can actually reconstruct.
- Made hard rollback best-effort on the repository side only after the logical rollback is durable in `run.json` and `journal.ndjson`, preserving the required ordering invariant even when `git reset --hard` fails.
- Hardened filesystem NDJSON appends to insert a separator newline when the existing file lacks a trailing newline, preventing rollback journal appends from corrupting existing history files.

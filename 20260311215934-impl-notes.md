# Implementation Notes

## Decisions Made
- Added `BackendFamily`, `BackendSpec`, `ModelSpec`, `ResolvedBackendTarget`, `BackendRole`, and `SessionPolicy` in [`src/shared/domain.rs`](/root/new-ralph-burning/ralph-burning-rewrite/src/shared/domain.rs) so backend selection and capability metadata stay in shared vocabulary instead of being redefined inside adapters or CLI code.
- Implemented `BackendResolver`, `AgentExecutionService`, `RawOutputPort`, `SessionManager`, and `SessionStorePort` across [`src/contexts/agent_execution/service.rs`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/service.rs), [`src/contexts/agent_execution/model.rs`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/model.rs), and [`src/contexts/agent_execution/session.rs`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/session.rs) so the facade owns preflight checks, timeout/cancellation handling, raw-output preservation, schema/domain validation, and session reuse metadata.
- Kept raw backend output ephemeral by introducing filesystem-backed `FsRawOutputStore` and `FsSessionStore` in [`src/adapters/fs.rs`](/root/new-ralph-burning/ralph-burning-rewrite/src/adapters/fs.rs), which write to `runtime/backend/` and `sessions.json` under the canonical project root without touching durable history artifacts.
- Added a deterministic `StubBackendAdapter` in [`src/adapters/stub_backend.rs`](/root/new-ralph-burning/ralph-burning-rewrite/src/adapters/stub_backend.rs) that emits stage-family-compatible JSON for every built-in stage, exposes configurable availability/capability/delay knobs, and supports session reuse behavior for contract and service tests.
- Added a CLI-level construction hook in [`src/cli/run.rs`](/root/new-ralph-burning/ralph-burning-rewrite/src/cli/run.rs) so the execution service is constructible from the current adapter wiring even before `run start` is implemented in loop 6.

## Spec Deviations
- `AgentExecutionPort::invoke` returns an `InvocationEnvelope` carrying an inline raw-output reference first, and `AgentExecutionService` converts that into a stored `runtime/backend/*` reference after persistence. This keeps the trait shape requested by the spec while still letting the service own raw-output preservation and final envelope normalization.

## Testing
- Verified resolver precedence, capability/availability failures, transport/schema/domain failure mapping, timeout handling, cancellation propagation, envelope construction, session policy enforcement, and raw-output/session persistence in [`tests/unit/agent_execution_test.rs`](/root/new-ralph-burning/ralph-burning-rewrite/tests/unit/agent_execution_test.rs).
- Verified `StubBackendAdapter` satisfies the `AgentExecutionPort` contract for every stage contract in [`tests/unit/stub_backend_test.rs`](/root/new-ralph-burning/ralph-burning-rewrite/tests/unit/stub_backend_test.rs).

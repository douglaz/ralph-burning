# Implementation Notes

## Decisions Made
- Added a separate `RunStatusJsonView` instead of changing text-mode `RunStatusView`, so `run status` kept its human-readable output while `run status --json` got the stable snake_case contract from the spec.
- Built `run history --json` with dedicated JSON formatting helpers rather than serializing raw payload/artifact records directly, so compact mode can omit heavy `payload`/`content` fields and verbose mode can add them back deterministically.
- Kept payload/artifact inspection visibility-safe by validating against visible journal history first and only then doing direct file reads by ID from `FsArtifactStore`.
- Implemented stage filtering and last-N tail slicing in the query layer so both CLI formatting and service-level tests use the same canonical selection logic.
- Added `docs/cli-reference.md` because the repo did not already contain the CLI reference file named in the spec.

## Spec Deviations
- The repo’s CLI integration test surface is consolidated in `tests/cli.rs`, so the Slice 4 CLI coverage landed there instead of a new `tests/cli/run_test.rs`.
- The repo’s full `cargo test --features test-stub` suite still fails in an unrelated existing conformance case, `daemon.pr_review.transient_error_preserves_staged`; Slice 4 verification was completed with targeted run-query unit tests, CLI tests, and `SC-RUN-001` through `SC-RUN-046` conformance runs.

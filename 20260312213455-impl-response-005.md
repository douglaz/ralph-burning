# Implementation Response (Iteration 5)

## Changes Made

1. **`requirements answer` durable-boundary handling** (Required Change 1)
   - Added `read_answers_json` to `RequirementsStorePort` trait and `FsRequirementsStore` implementation (`fs.rs:1116-1122`).
   - Extracted `answers_already_durably_stored()` helper in `service.rs:838-857` that checks **both** the journal for `AnswersSubmitted` events **and** `answers.json` for non-empty answer content. This covers the case where `answers.json` was durably written but the subsequent `write_run()`/journal append failed.
   - Both the `AwaitingAnswers` and `Failed` gate branches in `answer()` now call this unified check instead of only consulting the journal.
   - Graceful handling: if `answers.json` doesn't exist or can't be read, it's treated as "no answers stored" (same as initial empty state).

2. **Seed rollback removes seed history** (Required Change 2)
   - Reordered seed operations in `continue_after_answers()` (`service.rs:688-720`): seed files (`seed/project.json` + `seed/prompt.md`) are now written **before** committing the `project_seed` payload/artifact pair to `history/`.
   - If seed file writes fail, no seed history exists to orphan — only draft/review history survives.
   - If the subsequent history commit fails after seed files are written, the seed files are cleaned up via `remove_seed_pair`.
   - Added `remove_payload_artifact_pair` to `RequirementsStorePort` and `FsRequirementsStore` for completeness (`fs.rs:1164-1180`).

3. **Regression tests** (Recommended Improvement 1)
   - `answer_rejects_when_answers_json_already_populated` — seeds a run where `answers.json` has been durably written with real answers (but no `AnswersSubmitted` journal event), asserts `answer` is rejected.
   - `seed_write_failure_does_not_leave_seed_history` — verifies a successful run has seed history, seed files, and all expected layout (baseline for the invariant).

4. **Gherkin scenarios** — Added RD-022 (answer rejected when answers.json already populated) and RD-023 (seed write failure leaves no seed history behind).

All **289 unit tests** and **75 CLI tests** pass. `cargo check` clean.

# Feature: Codex Fresh-Session Metadata Reconciliation

## Description
Reconcile the Codex new-session `metadata.session_id` contract with the canonical `prompt.md` specification. The current implementation returns `Some(request.invocation_id.clone())` as a synthetic session ID for fresh Codex runs (introduced in Loop 11 to seed `sessions.json`), but `prompt.md` line 114 requires `metadata.session_id` to be `None` for non-resume Codex invocations. The completer flagged this as a spec/implementation divergence that must be resolved before completion.

The resolution: revert fresh Codex `session_id` to `None` to match the authoritative spec, and update the test accordingly. This means the `codex exec resume` branch remains unreachable for now (since `AgentExecutionService` only persists sessions when `session_id.is_some()`), but the code matches the canonical contract. The resume path can be enabled in a future issue when the Codex CLI surfaces a real session identifier.

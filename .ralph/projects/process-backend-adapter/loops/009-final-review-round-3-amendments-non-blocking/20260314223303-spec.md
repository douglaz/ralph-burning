# Feature: Final Review Round 3 Amendments — Non-Blocking Cancellation and Round-Aware Invocation IDs

## Description
Address the two remaining final-review amendments from Round 3. First, make `ProcessBackendAdapter::cancel()` non-blocking so that a child that ignores `SIGTERM` cannot stall the authoritative timeout/cancellation path. Second, include `completion_round` in workflow invocation IDs to prevent collisions across completion-round restarts that reuse the same cycle and attempt numbers.

## Acceptance Criteria
- [ ] `cancel()` returns promptly (within a bounded grace period) even when the child ignores `SIGTERM`; a background reap task handles lingering processes
- [ ] `kill_on_drop(true)` is set on spawned children as a safety net
- [ ] A regression test with a SIGTERM-ignoring fake backend proves `cancel()` returns within a bounded time
- [ ] Workflow invocation IDs include `completion_round`, using the format `{run_id}-{stage_id}-c{cycle}-a{attempt}-cr{completion_round}`
- [ ] Codex temp-file paths and raw-output persistence remain unique across completion rounds
- [ ] A test verifies that invocation IDs are distinct when only `completion_round` differs
- [ ] `nix develop -c cargo build` passes
- [ ] `nix develop -c cargo test` passes

## Files to Modify/Create
- `src/adapters/process_backend.rs` — Make `cancel()` non-blocking: remove the child from the active map, send `SIGTERM`, await `wait()` with a short bounded timeout (e.g. 500ms), and if the child hasn't exited, spawn a detached `tokio::spawn` task that escalates to `SIGKILL` and reaps. Set `kill_on_drop(true)` on `Command` construction. Remove the unbounded `wait()` call from the cancel path.
- `src/contexts/workflow_composition/engine.rs` — Change invocation ID format at ~line 1957 from `{run_id}-{stage_id}-c{cycle}-a{attempt}` to `{run_id}-{stage_id}-c{cycle}-a{attempt}-cr{completion_round}`, sourcing `completion_round` from `cursor.completion_round`.
- `tests/unit/process_backend_test.rs` — Add `cancellation_returns_promptly_when_child_ignores_sigterm()`: use a fake backend script that traps SIGTERM and sleeps forever, call `cancel()`, and assert it returns within 2 seconds. Add a test verifying `kill_on_drop(true)` behavior (child dies when handle is dropped).
- `tests/unit/workflow_engine_test.rs` — Add `invocation_ids_differ_across_completion_rounds()`: construct two `InvocationRequest`s for the same run/stage/cycle/attempt but different `completion_round` values and assert their `invocation_id` fields are distinct.

## Dependencies
- Requires: Loop 7 (Codex Resume Argv and Cancellation Child Reaping) — the current cancel/reap infrastructure being amended
- Blocks: none

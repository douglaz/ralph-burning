# Feature: Final Review Round 2 Amendments — Codex Resume Argv and Cancellation Child Reaping

## Description
Implements the two accepted-but-unimplemented amendments from the Loop 6 final review (Round 2). PBA-REV-001 fixes a runtime-blocking Codex resume argv mismatch where `--output-schema` is incorrectly passed to `codex exec resume`, which rejects it. PBA-REV-002 replaces bare-PID tracking with actual `tokio::process::Child` handle storage so that cancellation deterministically reaps the subprocess instead of leaving zombie/orphan processes.

## Acceptance Criteria
- [ ] Codex new-session command includes `--output-schema <path>` and `--output-last-message <path>`
- [ ] Codex resume command includes `--output-last-message <path>` but does NOT include `--output-schema <path>`
- [ ] Codex resume command shape is `codex exec resume --dangerously-bypass-approvals-and-sandbox --skip-git-repo-check --model <model_id> --output-last-message <message_path> <session_id> -`
- [ ] `ProcessBackendAdapter.active_children` stores `tokio::process::Child` handles (or a wrapper containing the handle), not bare `u32` PIDs
- [ ] `cancel()` sends SIGTERM to the child process AND awaits the child to exit (deterministic reap) before removing from the active-child map
- [ ] If the invocation is already gone from the map, `cancel()` returns `Ok(())`
- [ ] `kill_on_drop(true)` is set on spawned children as a safety net
- [ ] `spawn_and_wait()` registers the child handle in the active-child map before starting I/O, and removes it after completion
- [ ] State invariant: after `cancel()` returns, the child process is guaranteed to have exited (no zombie/orphan). If SIGTERM delivery fails because the process already exited, that is not an error.
- [ ] State invariant: if `cancel()` races with normal completion in `spawn_and_wait()`, whichever path removes the child first wins; the other path must handle the missing entry gracefully without panicking.
- [ ] New unit test: Codex resume command construction verifies `--output-schema` is absent from resume argv
- [ ] New unit test: Codex new-session command construction verifies `--output-schema` is present
- [ ] New unit test: cancellation test proves child is reaped (process no longer exists) after `cancel()` returns
- [ ] `nix develop -c cargo build` passes
- [ ] `nix develop -c cargo test` passes

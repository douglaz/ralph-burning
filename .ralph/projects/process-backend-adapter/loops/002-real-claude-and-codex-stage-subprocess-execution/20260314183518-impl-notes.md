# Implementation Notes

## Decisions Made
- **File-based fake CLI approach**: Test scripts read envelope/payload JSON from pre-written files via `cat`, avoiding shell escaping issues with embedded JSON
- **SIGTERM via `kill` command**: Used `Command::new("kill")` instead of `libc::kill` because `#![forbid(unsafe_code)]` is set project-wide; non-zero kill exit (process already gone) is treated as `Ok()`
- **Stdout/stderr ownership**: Took stdout/stderr handles before registering child in `active_children`, then spawned async read tasks concurrently with `child.wait()` — avoids `wait_with_output()` ownership conflict while keeping the child registered for cancel
- **Added tokio features**: `fs` and `io-util` for `tokio::fs` (Codex temp file I/O) and `AsyncReadExt`/`AsyncWriteExt` (stdin/stdout piping)
- **Session rules match spec exactly**: Claude prefers envelope `session_id` with prior session fallback; Codex sets `session_id` only on `exec resume`
- **Cancel always cleans `active_children`**: Entry removed immediately after signal attempt, preventing stale entries

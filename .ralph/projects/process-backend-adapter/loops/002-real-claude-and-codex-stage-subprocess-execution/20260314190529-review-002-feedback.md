# Review: SUGGESTIONS

## Required Changes
1. **Process lifecycle tracking**
Current: [process_backend.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/adapters/process_backend.rs#L129) awaits `stdin.write_all()` and `shutdown()` before the spawned child is inserted into `active_children` at [process_backend.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/adapters/process_backend.rs#L139). If `AgentExecutionService` times out or the cancellation token fires in that window, `cancel()` will see no registered child and can leave the subprocess running.
Expected: register the child immediately after `spawn()` and before any async stdin/stdout/stderr work, so every post-spawn cancellation race can still deliver SIGTERM and clean up tracking.
Reference: Feature spec acceptance criterion "State invariant on process lifecycle"; Master Prompt Required Changes 2 and 6.

2. **Codex temp artifact cleanup on early failure**
Current: [process_backend.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/adapters/process_backend.rs#L322) returns on schema-write failure without calling cleanup. Since file creation happens inside the write path, a partial `<invocation_id>.schema.json` can be left behind under `runtime/temp`, which violates the temp-artifact invariant.
Expected: run the same best-effort cleanup on schema-file creation/write failures, and keep cleanup failures non-fatal so the primary invocation error is preserved.
Reference: Feature spec acceptance criterion "State invariant on temp artifacts"; Master Prompt Required Changes 5 and 6.

## Recommended Improvements
1. Add regression tests that force cancellation during blocked stdin piping and force a Codex schema-write failure after file creation. The current tests cover wait/cancel and happy-path cleanup, but not these two failure windows.
2. Complete `nix develop -c cargo build` and `nix develop -c cargo test` verification in CI or a warmed Nix shell. `cargo` was not available on the base PATH here, and the Nix-shell runs did not finish during this review.

# Implementation Response (Iteration 1)
## Changes Made
Both required changes are implemented and verified:

1. **Cancel-safety**: `active_children` now stores PIDs (`u32`) instead of `Child` objects. The `Child` stays local to `spawn_and_wait()`, so `wait()` runs without holding the mutex. `cancel()` drops the lock before sending `SIGTERM` (now synchronous). No `await` crosses a lock boundary in either path.

2. **Invoke-path capability enforcement**: `invoke()` now calls `check_capability()` upfront, so `Requirements` contracts and unsupported families (`OpenRouter`, `Stub`) return `CapabilityMismatch` before any subprocess is spawned.

All 479 tests pass, including 2 new regression tests for the invoke-path fix.

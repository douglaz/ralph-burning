# Feature: Final Review Amendments â€” Worktree Threading, Rollback-Aware Prompts, Concurrent Subprocess I/O

## Description
Addresses three bugs found during final review of the ProcessBackendAdapter implementation:

1. **PBACK-REVIEW-001**: Daemon-dispatched workflow runs execute subprocesses in the shared repo checkout instead of the leased worktree. The `worktree_path` obtained from the lease is passed into `dispatch_in_worktree()` but the parameter is prefixed with `_` and discarded â€” `base_dir` is used for `InvocationRequest.working_dir` instead. This means daemon-spawned `claude`/`codex` processes edit the main checkout, defeating worktree isolation.

2. **PBACK-REVIEW-002**: `load_prior_stage_outputs_this_cycle()` reads the raw append-only `journal.ndjson` without applying rollback boundaries. The codebase already provides `visible_journal_events()` in `queries.rs` which filters events hidden by `RollbackPerformed` entries, but the prompt builder bypasses it. After a rollback or completion-round restart, discarded-branch outputs leak into later stage prompts.

3. **PBACK-REVIEW-003**: `spawn_and_wait()` writes the full stdin payload before spawning tasks to drain stdout/stderr. If the child fills its stdout pipe buffer before consuming all of stdin, both processes deadlock â€” the child blocks on stdout write while the parent blocks on stdin write. stdin writing, stdout reading, and stderr reading must happen concurrently.

# Feature: Workflow Stage Prompt Enrichment

## Description
Replace the placeholder `"Execute stage: {stage}"` prompt in `engine.rs` with a real prompt builder that assembles a rich, stage-aware prompt for subprocess backends. The prompt builder loads the original project prompt file, injects a role instruction for the current stage, recovers prior same-cycle stage outputs from the journal and payload history, includes remediation context and pending amendments when applicable, and appends the authoritative JSON schema for the stage contract. This is master-prompt requirement 9 — the last remaining feature.

## Acceptance Criteria
- [ ] A `build_stage_prompt` helper function exists in `engine.rs` (or a dedicated submodule) that constructs the full prompt string
- [ ] The prompt includes the original project prompt text read from `<project_root>/<project_record.prompt_reference>`
- [ ] The prompt includes a concise role instruction derived from `BackendRole` and `StageId` (e.g. "You are the Planner. Your objective for this Planning stage is…")
- [ ] The prompt includes a "Prior Stage Outputs This Cycle" section when earlier stages in the same `run_id` and `cycle` have completed, with payloads recovered by reading `journal.ndjson` for `stage_completed` events matching the current `run_id` and `cursor.cycle`, then loading the referenced `payload_id`s from `history/payloads/` via `ArtifactStorePort`, preserving journal order
- [ ] The prompt does NOT identify current-cycle payloads by scanning `history/payloads/` alone — journal is the source of truth for ordering and filtering
- [ ] The prompt includes a "Remediation / Pending Amendments" section when `execution_context` or `pending_amendments` are present
- [ ] The prompt includes a pretty-printed JSON schema section from `StageContract::json_schema()` with a note that the schema is authoritative and the response must conform to it
- [ ] `payload.context` remains populated via the existing `invocation_context()` call — prompt enrichment is additive, not a replacement for context plumbing
- [ ] `execute_run_internal` and `execute_stage_with_retry` are updated to thread `artifact_store: &dyn ArtifactStorePort` and `prompt_reference: &str` to the prompt builder call site
- [ ] All callers of `execute_run_internal` are updated to pass the new parameters
- [ ] If reading the project prompt file fails, the prompt builder returns an `AppError` (not a panic), so the stage fails gracefully with a clear message
- [ ] If recovering prior-cycle payloads fails (missing payload record referenced by journal), the prompt builder returns an `AppError` with a diagnostic message identifying the missing `payload_id`
- [ ] A prompt-builder test verifies that the assembled prompt contains: project prompt text, role instruction, prior same-cycle outputs (when present), remediation section (when present), amendments section (when present), and pretty-printed schema text
- [ ] A prompt-builder test verifies that when no prior stages have completed in the current cycle, the "Prior Stage Outputs" section is omitted
- [ ] A prompt-builder test verifies that remediation and amendments sections are omitted when those inputs are `None`/empty
- [ ] `nix develop -c cargo build` passes
- [ ] `nix develop -c cargo test` passes

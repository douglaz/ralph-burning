# Feature: Completion-round resume integrity after journal append failure

## Description
When a late-stage review triggers a completion-round advancement, the `completion_round_advanced` journal append may fail. Currently, the failure path persists the snapshot with pending amendments but WITHOUT the advanced `completion_rounds` counter. On resume, `derive_resume_state` correctly detects pending amendments and resets to the planning stage, but reconstructs the cursor with the OLD completion round (because neither the journal event nor the snapshot reflect the advancement). This causes the resumed run to generate history IDs with `-cr1` instead of `-cr2`, overwriting round-1 payload/artifact files and violating the append-only history invariant established in Loop 2.

The fix advances `snapshot.completion_rounds` **before** the `completion_round_advanced` journal append, so that if the append fails, `fail_run` writes a snapshot that already reflects the new round. Since `derive_resume_state` initializes `current_completion_round` from `snapshot.completion_rounds.max(1)` (engine.rs:2474), the resumed run will use the correct round for history IDs.

## Acceptance Criteria
- [ ] `snapshot.completion_rounds` is updated to `to_round` before the `completion_round_advanced` journal append at engine.rs:1028, so that the `fail_run` snapshot write captures the advanced round
- [ ] After a `completion_round_advanced` append failure, the persisted snapshot has `completion_rounds == to_round` and `amendment_queue.pending` populated
- [ ] On resume from that failure, `derive_resume_state` reconstructs a cursor with `completion_round == to_round`, not the pre-advance value
- [ ] History files written during the resumed run use `-cr{to_round}` (e.g. `-cr2`) and do not overwrite any `-cr1` files from the original round
- [ ] The normal (non-failure) code path is unaffected: the `.max(to_round)` assignment at the current line 1055 becomes a no-op since the value was already set, and the subsequent snapshot write at line 1067 still persists the full active-run state

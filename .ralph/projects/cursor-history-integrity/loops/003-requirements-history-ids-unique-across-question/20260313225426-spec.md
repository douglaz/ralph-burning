# Feature: Requirements history IDs unique across question rounds

## Description
Replace hardcoded `-1` suffixes in requirements drafting history IDs (question-set, draft, review) with the actual question round number. This ensures that if a requirements run progresses through multiple question rounds, each round's payload and artifact files get distinct IDs instead of overwriting the previous round's files. Seed IDs remain unchanged at `-1`.

## Acceptance Criteria
- [ ] Question-set payload IDs use `format!("{run_id}-qs-{round}")` where `round` is `run.question_round + 1` (the round being generated, since `question_round` is still `0` or the previous value at generation time)
- [ ] Question-set artifact IDs use `format!("{run_id}-qs-art-{round}")` with the same round
- [ ] Draft payload IDs use `format!("{run_id}-draft-{round}")` where `round` is the already-incremented `run.question_round`
- [ ] Draft artifact IDs use `format!("{run_id}-draft-art-{round}")` with the same round
- [ ] Review payload IDs use `format!("{run_id}-review-{round}")` where `round` is the already-incremented `run.question_round`
- [ ] Review artifact IDs use `format!("{run_id}-review-art-{round}")` with the same round
- [ ] Seed IDs remain unchanged: `format!("{run_id}-seed-1")` and `format!("{run_id}-seed-art-1")`
- [ ] `continue_after_answers` called from `quick()` (which skips questions) uses round `1` for draft/review IDs since `question_round` remains `0` — the method must compute the effective round as `max(run.question_round, 1)`
- [ ] `continue_after_answers` called from `draft()` with empty questions (no-question fast path at line 255) also correctly uses round `1`
- [ ] No changes to `run.json` field names, journal schema, or CLI commands/output structure
- [ ] No file renames or migration of existing files on disk
- [ ] Unit test expectations that hardcode `-qs-1`, `-draft-1`, or `-review-1` are updated to the round-aware format
- [ ] CLI test fixture at `tests/cli.rs:3815` that references `{run_id}-qs-1` is updated
- [ ] At least one regression test where `question_round == 2` asserts that draft and review IDs use `-2`
- [ ] All three verification commands pass: `cargo build`, `cargo test`, `conformance run`

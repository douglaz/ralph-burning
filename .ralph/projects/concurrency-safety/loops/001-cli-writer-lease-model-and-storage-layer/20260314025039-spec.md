# Feature: CLI Writer Lease Model and Storage Layer

## Description
Introduce a `CliWriterLease` record type and a `LeaseRecord` enum that distinguishes CLI writer leases from worktree leases at the data-model and storage layers. This is the foundation that all subsequent CLI lease work (guard, heartbeat, reconcile) builds on. The current `WorktreeLease` struct requires `task_id` and `worktree_path`, making it unsuitable for CLI-held locks. A new `CliWriterLease` struct carries only the fields relevant to CLI locks (no task ID, no worktree path), and a `LeaseRecord` enum wraps both variants for unified storage. Existing worktree lease files must continue to deserialize without modification (backward compatibility via serde defaults or internally-tagged representation with a fallback).

## Acceptance Criteria
- [ ] A new `CliWriterLease` struct exists in `model.rs` with fields: `lease_id`, `project_id`, `owner` (string, e.g. `"cli"`), `acquired_at`, `ttl_seconds`, `last_heartbeat`. No `task_id`, `worktree_path`, or `branch_name` fields.
- [ ] A `LeaseRecord` enum exists with variants `Worktree(WorktreeLease)` and `CliWriter(CliWriterLease)`, using `#[serde(tag = "lease_kind")]` internally-tagged representation.
- [ ] `CliWriterLease` implements `heartbeat_deadline()` and `is_stale_at()` with the same semantics as `WorktreeLease`.
- [ ] `LeaseRecord` exposes accessor methods for common fields: `lease_id()`, `project_id()`, `acquired_at()`, `is_stale_at()`.
- [ ] Existing serialized `WorktreeLease` JSON files (which lack `lease_kind`) deserialize correctly as `LeaseRecord::Worktree` via a serde default or untagged fallback.
- [ ] Newly serialized `LeaseRecord` files include the `lease_kind` discriminator field so they round-trip unambiguously.
- [ ] `DaemonStorePort` gains new methods: `list_lease_records()`, `read_lease_record()`, `write_lease_record()` that operate on `LeaseRecord`. The existing `list_leases()`, `read_lease()`, `write_lease()` methods remain unchanged (no breaking change) so that existing `LeaseService` code continues to compile and work.
- [ ] `FsDaemonStore` implements the new port methods, reading from and writing to the same `daemon/leases/` directory.
- [ ] `LeaseCleanupFailure.task_id` is changed to `Option<String>` (CLI leases have no task), with existing code updated to wrap the value in `Some(...)`.
- [ ] Unit tests prove: (a) `CliWriterLease` serde round-trip, (b) legacy `WorktreeLease` JSON (no `lease_kind` field) deserializes as `LeaseRecord::Worktree`, (c) `CliWriterLease` staleness detection works identically to `WorktreeLease`, (d) `list_lease_records()` returns both worktree and CLI lease records from the same directory.

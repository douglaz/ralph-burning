# Implementation Notes

## Decisions Made
- In reconcile pass 2, when `WriterLockReleaseOutcome::AlreadyAbsent` occurs, the code now proceeds to lease-file deletion instead of skipping it via `continue`. A new `writer_lock_absent` flag gates the early-exit: only non-absent lock failures (owner mismatch, I/O error) skip lease deletion. This ensures stale CLI lease records are pruned even when the writer lock was already released by `close()`, preventing rediscovery on subsequent reconcile runs.
- The `writer_lock_absent` path still sets `has_sub_step_failure = true`, so the lease is never counted in `released_lease_ids` — it remains a cleanup failure as required by the fail-closed accounting model.
- The worktree reconcile `Err(e)` arm for `LeaseService::release()` was relabeled from `worktree_remove` to `release_setup` because this path is reached when `release()` itself returns `Err` (e.g. `ProjectId::new()` validation), not when physical worktree removal fails. The `worktree_remove` label at line 453 (inside `outcome.worktree_error`) is unchanged since it genuinely refers to worktree removal I/O errors.
- The stable conformance binary now uses the test binary's parent directory (`target/debug/`) instead of `tempdir()`. This removes the dependency on an executable `/tmp`. A deterministic filename (`ralph-burning-stable-conformance`) is used with pre-cleanup to handle stale copies from previous runs, and post-cleanup after the conformance run completes.

# Implementation Response (Iteration 1)

## Changes Made
1. **Writer-lock release TOCTOU fix** (`src/adapters/fs.rs`): Rewrote `release_writer_lock` to open the lock file via fd, read contents through the fd, then compare inodes (via `fstat` on the fd vs `stat` on the path) before unlinking. If the inode changes between the content read and the unlink attempt, release fails closed with `OwnerMismatch { actual_owner: "(replaced)" }`. Added a `#[cfg(test)]` post-read hook (`release_lock_testing` module) so unit tests can inject a file replacement into the exact TOCTOU window.

2. **TOCTOU regression test** (`src/adapters/fs.rs` unit tests): Added `release_writer_lock_detects_replaced_file_via_inode` which uses the post-read hook to atomically replace the lock file (delete + recreate with different owner) between the content verification and the inode comparison. Asserts the release returns `OwnerMismatch` and the replacement file survives. This test lives in the library's own `#[cfg(test)]` module because the hook is only available to within-crate tests.

3. **Double-reporting fix** (`src/contexts/automation_runtime/lease_service.rs`): Changed the `OwnerMismatch` arm in `release()` to set `writer_lock_owner_mismatch = true` with `writer_lock_error = None` (was `Some(format!(...))`) so worktree reconcile no longer emits two `cleanup_failures` entries for a single owner-mismatch event.

4. **Missing-lease-file test fix** (`tests/unit/automation_runtime_test.rs`): Rewrote `reconcile_stale_cli_lease_missing_lease_file_reports_cleanup_failure` to use `SubStepAbsentStore::new(true, false)` instead of pre-deleting the lease file. The lease file now exists during `list_lease_records` (so reconcile discovers it as stale) but `remove_lease` returns `AlreadyAbsent` (simulating the disappearance race). The test now exercises the actual code path it describes and asserts `stale_leases == 1`, `released_leases == 0`, and cleanup failure with `lease_file_absent`.

# Implementation Response (Iteration 4)

## Changes Made
1. **Per-owner staging path eliminates shared-staging race** (`src/adapters/fs.rs`): Changed the staging target from the fixed `writer-<project>.lock.releasing` to `writer-<project>.lock.releasing.<expected_owner>` (with `/` replaced by `_` for path safety). Since each releaser passes a unique `expected_owner` (the lease_id, e.g. `cli-<uuid>`), concurrent releasers now stage to distinct paths. This closes the race where releaser B could rename its validated lock onto releaser A's staging file before A's final `remove_file()`, causing A to unknowingly delete B's lock.

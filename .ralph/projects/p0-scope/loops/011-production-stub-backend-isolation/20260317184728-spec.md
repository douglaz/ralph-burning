# Feature: Production Stub Backend Isolation

## Description
The shared `build_backend_adapter()` function in `src/composition/agent_execution_builder.rs` currently accepts `RALPH_BURNING_BACKEND=stub` and constructs a `StubBackendAdapter` at runtime. This builder is called from all three production entry points (`cli/run.rs`, `cli/requirements.rs`, `cli/daemon.rs`), violating the prompt's non-negotiable rule: "Production code must not directly instantiate `StubBackendAdapter` or file-based GitHub watchers. Those are test-only." This feature gates the `Stub` variant, all stub construction helpers, and the `"stub"` branch of the backend selector behind a Cargo feature flag `test-stub`, so that `cargo build` (production) never includes a stub backend path while `cargo test --features test-stub` continues to enable it for the full integration and conformance test suite.

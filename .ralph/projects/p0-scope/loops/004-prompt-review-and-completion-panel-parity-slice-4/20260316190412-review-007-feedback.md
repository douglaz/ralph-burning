## Required Changes
1. **Panel resolution and drift snapshots**: move required/optional availability filtering into the resolved panel state, before snapshot persistence and before resume drift comparison.
Current: [policy.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/policy.rs#L203) only filters on config-enabled backends, [engine.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L3685) and [engine.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L3912) persist snapshots from that unfiltered panel, and only later do [prompt_review.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/prompt_review.rs#L137) and [completion.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/completion.rs#L107) drop unavailable members. That means `stage_resolution_snapshot` can record panel members that never execute, and resume-drift/minimum-count checks can pass on members that are already unavailable.
Expected: required unavailable panel members should fail resolution before the stage starts, optional unavailable members should be removed before building/persisting the snapshot, and resume drift should compare the actual executable panel so availability-only drift warns or fails before any panel invocation.
Reference: Shared Foundations “Filtering rules”; Slice 4 acceptance criteria for exact `StageResolutionSnapshot` targets, prompt-review/completion filtering, and resume-drift early failure.

2. **Completion `continue_work` commit ordering**: do not make the aggregate and round-advance durable before the advanced cursor snapshot is durably written.
Current: [engine.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L1378) persists the aggregate, [engine.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L1402) appends `completion_round_advanced`, and only then [engine.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L1449) writes the next cursor snapshot. If that snapshot write fails, the aggregate and round event remain durable, and [engine.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L3537) resume reconstructs from planning instead of `completion_panel`.
Expected: if the post-panel cursor transition cannot be persisted, the run must stay resumable from `completion_panel`, with only supporting records durable and no durable aggregate/round-transition signal left behind.
Reference: Slice 4 acceptance criterion “Completion failure invariant”.

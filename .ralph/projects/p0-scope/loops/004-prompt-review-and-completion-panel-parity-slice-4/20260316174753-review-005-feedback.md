## Required Changes
1. **Panel backend availability and optional-skip semantics**: prompt-review and completion panels still do not implement the spec’s required/optional availability filtering against real backend availability.
Current: [policy.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/policy.rs#L196) only filters panel members with `backend_enabled`, and [policy.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/policy.rs#L326) treats “resolvable” as enabled-only. After that, [prompt_review.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/prompt_review.rs#L142) returns any validator invocation error directly, and [completion.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/completion.rs#L111) fails the stage on any completer invoke error. An optional backend that is enabled but actually unavailable at runtime therefore fails the stage instead of being skipped, and required backends are not rejected until invocation time.
Expected: resolve panel members with real availability/capability checks while preserving required vs optional semantics, skip optional unavailable members before execution, and fail required unavailable members before any panel invocation.
Reference: Shared Foundations “Filtering rules”, plus Slice 4 acceptance criteria for `workflow.prompt_review.optional_validator_skip`, `workflow.completion.optional_backend_skip`, and `workflow.completion.required_backend_failure`.

Expected: carry invocation metadata through the stage-success path and persist `RecordProducer::Agent { backend_family, model_id }` for primary non-panel stage records.
Reference: Shared Foundations “PayloadRecord and ArtifactRecord gain `record_kind`, `producer`, and `completion_round`”, and the Slice 4 acceptance criterion requiring history/tail to surface record-kind and producer metadata.

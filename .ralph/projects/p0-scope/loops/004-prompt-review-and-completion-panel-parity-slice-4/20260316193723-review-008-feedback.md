## Required Changes
1. **Resume drift warning durability**
Current: [engine.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L4421) treats a failed `durable_warning` journal append as non-fatal, then [engine.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L4442) updates the stored snapshot and continues. That means a drifted resume can proceed without the required durable warning evidence.
Expected: if the resume-drift warning cannot be appended, resume should fail before continuing. The spec requires one runtime warning and one durable journal warning before continuing with the new resolution.
Reference: Recovery and Failure Precedence, plus Slice 4 acceptance criterion for resume drift detection.

2. **Prompt-review refiner is not validated as part of panel resolution**
Current: [policy.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/policy.rs#L99) resolves a distinct prompt-review refiner target, but [engine.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L3752) only availability-filters validators before snapshot persistence, and [prompt_review.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/prompt_review.rs#L89) invokes the refiner after `stage_entered`. On start, [engine.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L205) preflights only the generic `StagePlan` target, not the actual panel refiner. The resume-drift path has the same validator-only filtering at [engine.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L763).
Expected: required prompt-review refiner resolution should include the same early availability/capability checks as the validator/completer panel members, so an unavailable refiner fails before stage start, before snapshot persistence, and before resume drift comparison declares the panel executable.
Reference: Shared Foundations filtering and general failure rules, plus Slice 4 prompt-review panel resolution requirements.

# Review: SUGGESTIONS

## Required Changes
1. **PR lifecycle after no-diff closure**: Closing a no-diff PR leaves `task.pr_url` populated, which blocks later draft-PR creation if the completed project is reopened and produces new changes.
   - Current: [pr_runtime.rs#L57](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/automation_runtime/pr_runtime.rs#L57) returns early from `ensure_draft_pr` on any stored `pr_url`; [pr_runtime.rs#L115](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/automation_runtime/pr_runtime.rs#L115) likewise treats any stored URL as the canonical PR during completion; [pr_runtime.rs#L229](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/automation_runtime/pr_runtime.rs#L229) through [pr_runtime.rs#L253](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/automation_runtime/pr_runtime.rs#L253) close the PR but never clear or invalidate that stored URL.
   - Expected: When `CloseOnNoDiff` closes an existing PR, clear the stored `pr_url` or detect closed-PR state and recreate a draft PR on the next ahead-of-base run. Otherwise reopened completed work can no longer satisfy the slice-9 PR lifecycle.
   - Reference: Slice 9 acceptance criteria for `When a task completes with no diff from base, PR creation is skipped or existing PR is closed per policy` and `If new review amendments arrive for a completed project, the project is reopened ... and remediation work is dispatched`. The reopen path is present in [pr_review.rs#L157](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/automation_runtime/pr_review.rs#L157) and is fed back into the same-cycle dispatcher at [daemon_loop.rs#L475](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/automation_runtime/daemon_loop.rs#L475), so the stale closed URL is a real production-path blocker.

## Recommended Improvements
1. Add an end-to-end conformance case for `close existing PR on no diff -> ingest PR review amendments -> reopen project -> create fresh draft PR after new diff`, because the current slice-9 scenarios cover close/create separately but do not catch this lifecycle break.

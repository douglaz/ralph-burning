## Required Changes
1. **Project-scoped daemon policy resolution**: The daemon still resolves slice-9 PR/rebase policy at the repo workspace level instead of per project, so project `config.toml` overrides are ignored.
   Current: [daemon_loop.rs#L539](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/automation_runtime/daemon_loop.rs#L539) uses `EffectiveConfig::load(repo_root)` for task execution, and [daemon_loop.rs#L1895](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/automation_runtime/daemon_loop.rs#L1895) loads the review whitelist the same way. But [config.rs#L152](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workspace_governance/config.rs#L152) only applies project overrides through `load_for_project(..., Some(project_id), ...)`, including `daemon.pr.*` and `daemon.rebase.*` in [config.rs#L320](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workspace_governance/config.rs#L320).
   Expected: Resolve effective config per task/project before PR review ingestion, completion PR handling, rebase policy, and rebase backend selection, so `daemon.pr.review_whitelist`, `daemon.pr.no_diff_action`, `daemon.rebase.*`, and project backend overrides all honor the required precedence.
   Reference: Slice 9 acceptance criteria for configurable review whitelist, no-diff PR policy, and config-gated rebase behavior; Master Prompt “Config precedence exactly: CLI overrides, Project config, Workspace config, Code defaults”.

2. **Completion-path PR cancellation**: The production completion path does not honor the task-level cancellation token, so an abort can still race into PR mutations and task completion after dispatch returns.
   Current: After the one abort-status check in [daemon_loop.rs#L705](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/automation_runtime/daemon_loop.rs#L705), the daemon calls `handle_completion_pr(..., &shutdown)` in [daemon_loop.rs#L724](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/automation_runtime/daemon_loop.rs#L724) and again in the single-repo path at [daemon_loop.rs#L1176](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/automation_runtime/daemon_loop.rs#L1176). But [pr_runtime.rs#L83](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/automation_runtime/pr_runtime.rs#L83) only checks the token it is given, via [pr_runtime.rs#L254](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/automation_runtime/pr_runtime.rs#L254).
   Expected: Carry the task cancellation token through completion PR handling, or re-check task abort state around PR push/create/close/ready transitions, so cancellation cleanly prevents post-dispatch PR side effects.
   Reference: Slice 9 acceptance criterion “PR runtime supports clean cancellation via the existing cancellation token”.

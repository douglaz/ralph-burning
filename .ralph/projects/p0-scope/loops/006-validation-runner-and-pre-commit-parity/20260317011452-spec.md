# Feature: Validation Runner and Pre-Commit Parity

## Description
Loops 1 through 5 are already completed and approved: config/backend policy, real requirements backend execution, OpenRouter support, prompt-review and completion panels, and final-review/prompt-drift/cap behavior. The current baseline also already has validation config fields, docs/CI flow stages, and snapshot-based rollback points, but Slice 6 is still missing: there is no `src/adapters/validation_runner.rs`, no `src/contexts/workflow_composition/validation.rs`, docs/CI validation still runs through agent-style validation payload paths, the Slice 6 conformance scenario IDs do not exist, and git history shows no later loop or `rb: checkpoint` work. The next logical feature is therefore real local validation execution plus pre-commit gating.

## Acceptance Criteria
- [ ] Add a real validation runner that executes configured command groups with `sh -lc` from the repo root, captures `command`, `exit_code`, `stdout`, `stderr`, `duration_ms`, and `passed`, enforces a per-command timeout, and never mutates run state directly.
- [ ] Return structured `ValidationCommandResult` and `ValidationGroupResult`; reuse an existing validation-timeout constant if one already exists, otherwise default to 900 seconds per command.
- [ ] Empty command lists are valid no-op passes for `standard_commands`, `docs_commands`, and `ci_commands`.
- [ ] Command groups execute in order and preserve partial evidence: if a command fails or times out, the group result is failed, completed commands plus the failing command are persisted in order, and later commands are not required to run.
- [ ] `docs_change` and `ci_improvement` validation stages become local-validation stages and must not invoke an agent/backend in P0.
- [ ] `docs_commands` and `ci_commands` map to `ValidationPayload` deterministically: all commands pass => `approved`; any command fails => `request_changes`, with evidence summaries, failing-command excerpts, and follow-up items derived from the structured results.
- [ ] Docs/CI validation failures reuse the existing remediation loop: `docs_validation` returns to `docs_update`, `ci_validation` returns to `ci_update`, and durable supporting evidence is written before any cycle transition.
- [ ] Standard flow continues to use agent QA and review stages, but review context must include local validation evidence generated from `validation.standard_commands`.
- [ ] Local validation evidence is persisted as supporting history records with `RecordKind::StageSupporting` and `RecordProducer::LocalValidation`, so it remains distinct from agent-produced primary stage output.
- [ ] After review approval in the standard flow, run pre-commit checks equivalent to old P0 behavior: `cargo fmt --check`, `cargo clippy --all-targets -- -D warnings`, and `nix build`, each controlled by the existing config booleans.
- [ ] Skip cargo-based pre-commit checks when the repo root does not contain `Cargo.toml`; `nix build` remains independently controlled by config.
- [ ] If `pre_commit_fmt_auto_fix = true` and `cargo fmt --check` fails, run `cargo fmt`, rerun `cargo fmt --check`, and count fmt as passed only if the rerun succeeds; both the original failure and repair attempt must be reflected in structured results.
- [ ] Any pre-commit failure invalidates the current reviewer approval, persists supporting validation evidence and remediation feedback, emits a runtime log entry, and returns the run to implementation remediation instead of allowing completion or QA to proceed.
- [ ] Failure invariant: on pre-commit failure, only reviewer approval is cleared; prompt-review, completion-panel, final-review, and prior-cycle durable history remain unchanged.
- [ ] Failure invariant: if command execution finishes but supporting payload/artifact or journal persistence fails afterward, the run does not advance, does not synthesize approval or remediation, and remains resumable from the pre-transition stage boundary.
- [ ] Add unit and conformance coverage for `validation.docs.commands_pass`, `validation.docs.command_failure_requests_changes`, `validation.ci.commands_pass`, `validation.ci.command_failure_requests_changes`, `validation.standard.review_context_contains_local_validation`, `validation.pre_commit.disabled_skips_checks`, `validation.pre_commit.no_cargo_toml_skips_cargo_checks`, `validation.pre_commit.fmt_failure_triggers_remediation`, `validation.pre_commit.fmt_auto_fix_succeeds`, and `validation.pre_commit.nix_build_failure_records_feedback`.
- [ ] Run the broad regression suite before closing the slice; if any full-suite prerequisite is unavailable, report the exact blocker and validate with the narrowest meaningful unit/conformance subset instead.

# Review: SUGGESTIONS

## Required Changes
1. **Review-context injection**: Local validation evidence is collected, but it never reaches the `review` stage invocation.
   - Current: [engine.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L2860) stores `standard_commands` output into `execution_context`, then [engine.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L2912) only forwards that context when `stage_id == semantics.execution_stage`, so `review` drops it.
   - Expected: The `review` stage prompt/context must receive the local validation evidence from `validation.standard_commands`.
   - Reference: Slice 6 acceptance criterion for standard flow review context, plus `validation.standard.review_context_contains_local_validation`.

2. **Local-validation stage isolation**: `docs_validation` and `ci_validation` still depend on backend preflight even though they are meant to be local-only stages.
   - Current: [engine.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L591) resolves a backend target for every stage and [engine.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L595) preflights them all before execution; only later does [engine.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L2440) branch into the local validation path.
   - Expected: `docs_validation` and `ci_validation` must not require agent/backend capability or availability checks in P0.
   - Reference: Slice 6 acceptance criterion stating docs/CI validation stages are local-validation stages and must not invoke an agent/backend.

3. **Durability/failure handling**: Supporting-evidence persistence failures are ignored, which breaks the slice’s failure invariant.
   - Current: [engine.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L2874) ignores failure from standard local-validation evidence persistence, and [engine.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L3047) ignores failure from pre-commit evidence persistence. The run can still advance without durable evidence/remediation feedback.
   - Expected: If supporting payload/artifact persistence fails after command execution, the run must stop at the pre-transition boundary and must not proceed with approval/remediation.
   - Reference: Slice 6 acceptance criteria for supporting evidence persistence and the explicit failure invariant.

# Review: SUGGESTIONS

## Required Changes
1. **Panel backend availability filtering**: Optional panel backends are only filtered by config enablement, not by actual availability/capability.
   - Current: [`resolve_panel_backends`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/policy.rs#L195) only checks `backend_enabled()`, so an enabled `?openrouter` is still resolved into the panel. That conflicts with the current adapter behavior, where [`ProcessBackendAdapter`](/root/new-ralph-burning/ralph-burning-rewrite/src/adapters/process_backend.rs#L563) still rejects OpenRouter capability/availability.
   - Expected: Optional backends must be skipped when they are disabled or unavailable, and required unavailable backends must fail resolution immediately.
   - Reference: Slice 1 acceptance criterion for panel backend filtering and the Shared Foundations filtering rules (`required-unavailable fails resolution`, `optional-unavailable silently skips`).

2. **Completion panel default alternation**: The default completion-panel behavior does not implement the required cycle-based completer family rule.
   - Current: [`resolve_role_target`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/policy.rs#L48) correctly defines the default `Completer` as the opposite family, but [`resolve_completion_panel`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/policy.rs#L74) ignores that and always uses `completion_policy().backends`. Those defaults are hard-coded to `claude` and `codex` in [`config.rs`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workspace_governance/config.rs#L1198), so an unconfigured completion panel can still include the same family as the completion planner.
   - Expected: When `completion.backends` is not explicitly configured, the default completer set should be derived from the cycle policy so the default completer family is opposite the completion planner.
   - Reference: Slice 1 acceptance criterion under default family alternation (`completion planner same as planner; default completer opposite of completion planner`).

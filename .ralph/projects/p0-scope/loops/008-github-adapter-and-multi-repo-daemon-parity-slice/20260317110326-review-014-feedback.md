# Review: SUGGESTIONS

## Required Changes
1. **Phase 0 recovery truthfulness**: the new label-repair recovery can hide retained daemon resources.
Current: [daemon_loop.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/automation_runtime/daemon_loop.rs#L286) clears `label_dirty` before recovery, then [task_service.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/automation_runtime/task_service.rs#L458) calls `LeaseService::release()` and ignores the result, always writing the task back as `pending` with `lease_id = None`. But [lease_service.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/automation_runtime/lease_service.rs#L249) treats partial cleanup as `resources_released = false`, so this path can drop the durable lease reference while cleanup is incomplete. The shipped conformance case in [scenarios.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/conformance_spec/scenarios.rs#L12464) currently asserts that incorrect outcome.
Expected: only revert to `pending` and clear `lease_id` after cleanup positively succeeds; otherwise keep the task in a recovery-visible state and preserve the lease/worktree ownership for later repair.

2. **Abort/retry command semantics**: command-driven task changes do not consistently reuse the existing cleanup path.
Current: GitHub `/rb abort` in [github_intake.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/automation_runtime/github_intake.rs#L291) only calls `mark_aborted()` plus label sync, unlike CLI abort in [daemon.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/cli/daemon.rs#L338) which runs `cleanup_aborted_task()` for `claimed` and `active` tasks. Separately, `retry_task()` in [task_service.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/automation_runtime/task_service.rs#L431) blindly clears `lease_id`, and both CLI retry [daemon.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/cli/daemon.rs#L395) and GitHub `/rb retry` [github_intake.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/automation_runtime/github_intake.rs#L279) use it. A task that retained its lease after a quarantined label failure or partial cleanup can therefore be aborted or retried into a state that strands or hides live resources.
Expected: route GitHub abort through the same cleanup/release behavior as the existing daemon abort path, and make retry fail or explicitly repair retained leases/worktrees instead of clearing the reference unconditionally.

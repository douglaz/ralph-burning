# Review: SUGGESTIONS

## Required Changes
1. **GitHub intake task creation is not atomic with the required repo metadata**: the current intake path can persist a live task that never gets `repo_slug`, `issue_number`, or comment cursor metadata.
   - Current: [`github_intake.rs`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/automation_runtime/github_intake.rs#L231) creates the task first, then patches GitHub metadata in a second `write_task()` at [`github_intake.rs`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/automation_runtime/github_intake.rs#L242). The initial record is written with `repo_slug: None` and `issue_number: None` at [`task_service.rs`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/automation_runtime/task_service.rs#L645). If that second write fails, later polls hit the idempotent `Ok(None)` path at [`task_service.rs`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/automation_runtime/task_service.rs#L609) and never repair the missing metadata.
   - Expected: create the task atomically with the GitHub fields already populated, or add a repair path that backfills missing GitHub metadata before the idempotent early return.
   - Reference: Slice 8 acceptance criteria for persisted GitHub task metadata during intake, and task identity / duplicate prevention keyed by `repo_slug + issue_number`.

2. **Repo bootstrap still exposes the GitHub token in process arguments**: the credential no longer lands in `.git/config`, but it is still passed on argv.
   - Current: [`repo_registry.rs`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/automation_runtime/repo_registry.rs#L322) spawns `git -c http.extraHeader=Authorization: Bearer <token> clone ...`. That avoids persistence, but the raw bearer token is visible to local process inspection while the clone runs.
   - Expected: inject auth through a non-persisted, non-argv mechanism such as env-backed Git config (`GIT_CONFIG_COUNT` / `GIT_CONFIG_KEY_*` / `GIT_CONFIG_VALUE_*`) or an equivalent askpass/header path.
   - Reference: Review criterion 2 (`Code quality - secure`), plus the slice-8 startup/bootstrap requirement.

# Implementation Response (Iteration 9)

## Changes Made
1. **RC1 — Remove legacy single-repo CLI branches**: `Status`, `Abort`, `Retry`, and `Reconcile` subcommands now require `--data-dir` (was `Option<String>`, now `String`). `Abort` and `Retry` also require `--repo`. The four legacy handler functions were deleted. All 12 affected CLI tests and 10+ conformance scenarios were converted to use data-dir layout with new test helpers.

2. **RC2 — Per-repo failure isolation for label sync**: `daemon_loop.rs` now quarantines a repo when resumed-task label sync fails, using `continue` to skip to the next repo instead of proceeding to phase 3. This matches the quarantine behavior already applied to poll failures and task-processing failures.

3. **RI1 — URL-encode refs in `is_branch_ahead()`**: Branch refs are now percent-encoded (`/` → `%2F`) before interpolation into the GitHub compare URL path, preventing malformed URLs for branch names like `rb/42-project-id`.

# Implementation Response (Iteration 21)

## Changes Made
1. **RC1 — Startup repo validation: parse and validate workspace config**: Strengthened `validate_repo_checkout()` in `repo_registry.rs` to call `load_workspace_config()` after verifying `workspace.toml` exists. A malformed workspace config (invalid TOML, missing required fields, wrong structure) is now caught at `daemon start` time instead of failing later during task processing.

2. **RC2 — Startup label ensure: fail explicitly on partial success**: Changed `run_multi_repo()` in `daemon_loop.rs` to compare active vs requested registrations after `ensure_labels_on_repos()`. If any requested repo was quarantined, the daemon returns an explicit startup error naming the failing repos instead of silently excluding them.

3. **RI1 — Extended conformance coverage**: 
   - `daemon.github.start_validates_repos_and_data_dir` now tests malformed `workspace.toml` rejection (both structurally invalid and broken TOML syntax).

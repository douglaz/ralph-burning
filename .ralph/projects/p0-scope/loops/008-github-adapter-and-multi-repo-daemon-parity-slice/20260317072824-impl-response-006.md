# Implementation Response (Iteration 6)

## Changes Made
1. **RC1 — GitHub-only production intake**: `daemon start` without `--data-dir` now always returns an error in normal CLI usage. The inconsistency between `github_intake.rs` (which accepted both) and `task_service.rs` (which only accepted `Failed`) is resolved.

3. **RI1 — Conformance coverage**: Added `daemon.tasks.retry_aborted_issue` and `daemon.tasks.start_requires_data_dir` conformance scenarios.

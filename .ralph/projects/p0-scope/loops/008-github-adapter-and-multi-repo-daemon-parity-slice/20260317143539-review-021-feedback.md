## Required Changes
1. **Startup repo validation**: reject unusable checkouts before the daemon begins polling.
   - Current: [`validate_repo_checkout()`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/automation_runtime/repo_registry.rs#L171) only verifies `.git`, `.ralph-burning/`, and that `workspace.toml` exists, and [`handle_start_multi_repo()`](/root/new-ralph-burning/ralph-burning-rewrite/src/cli/daemon.rs#L176) treats that as sufficient startup validation. A malformed workspace config is therefore accepted at startup; the runtime later either silently falls back to `FlowPreset::Standard` during intake at [`daemon_loop.rs`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/automation_runtime/daemon_loop.rs#L379) or fails only when a task is processed at [`daemon_loop.rs`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/automation_runtime/daemon_loop.rs#L495).
   - Expected: parse and validate the checkout’s workspace config during `daemon start` so an unusable repo fails early and explicitly, instead of being accepted and breaking later in the cycle.
   - Reference: slice-8 acceptance criterion for early repo validation in [`20260317041753-spec.md`](/root/new-ralph-burning/.ralph/projects/p0-scope/loops/008-github-adapter-and-multi-repo-daemon-parity-slice/20260317041753-spec.md#L25)

2. **Startup label ensure behavior**: do not silently quarantine requested repos during `daemon start`.
   - Current: [`run_multi_repo()`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/automation_runtime/daemon_loop.rs#L201) calls [`ensure_labels_on_repos()`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/automation_runtime/github_intake.rs#L433), and that helper explicitly logs and drops repos whose label ensure fails instead of returning an error. As shipped, `daemon start --repo a --repo b` can succeed with one requested repo silently excluded, as long as at least one repo survives.
   - Expected: if startup cannot ensure the required label vocabulary for a requested repo, surface that as an explicit startup failure for `daemon start` rather than silently skipping the repo.
   - Reference: slice-8 startup validation and label-ensure criteria in [`20260317041753-spec.md`](/root/new-ralph-burning/.ralph/projects/p0-scope/loops/008-github-adapter-and-multi-repo-daemon-parity-slice/20260317041753-spec.md#L25) and [`20260317041753-spec.md`](/root/new-ralph-burning/.ralph/projects/p0-scope/loops/008-github-adapter-and-multi-repo-daemon-parity-slice/20260317041753-spec.md#L28)

### Implementation Notes

**Amendment normalization function** (`normalize_amendment_body`):
1. Replace all `\r\n` and `\r` with `\n`
2. Trim leading/trailing whitespace
3. Collapse internal runs of whitespace (spaces, tabs) to single spaces (preserve `\n` boundaries)
4. Return the normalized string

**Amendment ID generation** (`canonical_amendment_id`):
1. Normalize the body
2. Compute SHA-256 of the normalized body
3. Format as `fr-{completion_round}-{sha256_hex[:8]}`

**State invariants for failure modes**:
- If a reviewer invocation fails but is required: the entire final-review stage fails. No partial voting occurs.
- If a reviewer invocation fails but is optional (and `min_reviewers` is still met after filtering): skip that reviewer, proceed with remaining proposals.
- If the arbiter invocation fails: the final-review stage fails. Disputed amendments cannot be resolved without the arbiter.
- If prompt-change `restart_cycle` clears supporting records: only records from the current cycle's abandoned stages are cleared; prior cycle history and aggregate records from completed stages remain intact.
- QA/review iteration counters reset to 0 at the start of each new cycle. `final_review_restart_count` persists across the run (not per-cycle).

**Conformance scenarios to implement**:
- `workflow.final_review.no_amendments_complete` - All reviewers propose no amendments; final review completes immediately
- `workflow.final_review.restart_then_complete` - Reviewers propose amendments, voting reaches consensus, restart with amendments, second pass completes
- `workflow.final_review.planner_completion_with_pending_amendments_fails` - Planner cannot skip pending amendments
- `workflow.final_review.disputed_amendment_uses_arbiter` - Mixed votes trigger arbiter for disputed amendment only
- `workflow.final_review.restart_cap_force_complete` - `max_restarts` exceeded triggers force-complete with explanatory artifact
- `workflow.resume.prompt_change_continue_warns` - Resume with changed prompt under `continue` policy emits warning and proceeds
- `workflow.resume.prompt_change_abort_fails` - Resume with changed prompt under `abort` policy fails with error
- `workflow.resume.prompt_change_restart_cycle` - Resume with changed prompt under `restart_cycle` policy resets cycle to planning
- `workflow.resume.backend_drift_warns` - Resume with changed backend resolution emits warning (extends existing drift tests to cover final-review panel)

If an optional reviewer succeeds in proposals but fails in voting, the whole final-review stage fails.
   - Expected: Carry optional/required membership through the whole panel flow and apply the same skip-and-recheck-`min_reviewers` behavior during voting that already exists during proposal collection.
   - Reference: Slice 5 implementation notes: optional reviewer invocation failures should be skipped when `min_reviewers` is still satisfied.

2. **Prompt-change restart-cycle cleanup scope**: `restart_cycle` only clears supporting records from the interrupted stage onward, not from the full abandoned portion of the cycle.
   - Current: [`evaluate_prompt_change_on_resume()`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/drift.rs#L94) calls cleanup with [`&stage_plan[current_stage_index..]`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/drift.rs#L115). If resume restarts from a late stage like `final_review`, supporting records from earlier abandoned panel stages in the same cycle, such as `completion_panel`, are left behind.
   - Expected: Clear supporting records for every stage invalidated by the cycle restart, starting from the flow’s planning stage for the current cycle rather than from the interrupted stage index.
   - Reference: Slice 5 acceptance criterion for `restart_cycle` clearing supporting records from the abandoned portion of the cycle.

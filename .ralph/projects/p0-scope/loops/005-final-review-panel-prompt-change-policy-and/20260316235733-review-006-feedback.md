# Review: SUGGESTIONS

## Required Changes
1. **Prompt-change continue baseline**: `continue` should warn and proceed without rewriting the cycle-start prompt baseline.
   - Current: `evaluate_prompt_change_on_resume()` returns the new prompt hash on `continue` in [drift.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/drift.rs#L71), and resume rebuilds the active run with that value as both `prompt_hash_at_cycle_start` and `prompt_hash_at_stage_start` in [engine.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L1148). That changes the baseline later resumes are supposed to compare against.
   - Expected: Keep `prompt_hash_at_cycle_start` anchored to the original cycle-start hash on `continue`; only `restart_cycle` should establish a new prompt baseline for resumed work.
   - Reference: Slice 5 acceptance criteria for `evaluate_prompt_change_on_resume()` and the prompt-change policy definition, which only calls for updating the prompt hash when `restart_cycle` is selected.

2. **Per-cycle QA/review counters**: QA and review iteration counters are being carried across cycle boundaries instead of resetting for a new cycle.
   - Current: after a QA/review-triggered remediation, the next cycle is seeded with incremented prior counts in [engine.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L2949), and resume reconstructs those counters by cumulatively scanning all historical `cycle_advanced` events in [engine.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L384) and [engine.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L1145). A later cycle can therefore hit the cap because of failures from earlier cycles.
   - Expected: Start each new cycle with `qa_iterations_current_cycle = 0` and `review_iterations_current_cycle = 0`; only `final_review_restart_count` should persist across cycle boundaries.
   - Reference: Slice 5 implementation notes: QA/review counters reset to `0` at the start of each new cycle, while `final_review_restart_count` persists across the run.

## Recommended Improvements
1. Add a resume regression where `continue` is used once, the run fails again in the same cycle, and a second resume still compares against the original cycle-start prompt hash.
2. Add a workflow regression where QA or review requested changes in a prior cycle, then a fresh cycle starts and proves the new cycle’s cap accounting begins at zero.

# Review: SUGGESTIONS

## Required Changes
1. **Final-review source metadata**: Preserve distinct reviewer sources when duplicate panel members resolve to the same backend/model.
Current: [final_review.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/final_review.rs#L128) deduplicates `sources` by full equality, but [final_review.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/final_review.rs#L268) synthesizes `reviewer_id` from only `<backend>:<model>`. Since [policy.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/policy.rs#L203) preserves duplicate panel specs, two separate reviewers on the same target collapse into one `FinalReviewAmendmentSource`.
Expected: Give each resolved reviewer a stable per-member identity, and use that in `FinalReviewAmendmentSource`, so merged amendments retain every contributing reviewer.
Reference: Slice 5 acceptance criterion: "Duplicate amendments merged by normalized-body hash; source reviewer metadata preserved."

2. **Prompt-change resume durability**: Do not mutate durable state before the prompt-change warning has been durably recorded.
Current: [drift.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/drift.rs#L67) updates `project.toml` for `continue`, and [drift.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/drift.rs#L91) clears supporting records and updates `project.toml` for `restart_cycle`, before [drift.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/drift.rs#L102) appends the durable warning. If that warning persistence fails, resume exits after partially applying the action, and a later resume can miss the required warning because the stored prompt hash was already advanced.
Expected: Make the durable warning the commit point, or roll back the prompt-hash/supporting-record mutations on failure, so `continue` and `restart_cycle` are atomic with respect to the required warning and restart semantics.
Reference: Slice 5 acceptance criteria for `continue` / `restart_cycle` and the `Recovery and Failure Precedence` section.

## Recommended Improvements
1. Add a unit or conformance case with duplicated final-review backends so source-preservation is locked in.
2. Add a failure-injection test around prompt-change warning persistence to prove there is no partial state mutation when the warning commit fails.

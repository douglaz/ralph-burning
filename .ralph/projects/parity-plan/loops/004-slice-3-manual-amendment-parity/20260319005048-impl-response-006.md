# Implementation Response (Iteration 6)

## Changes Made
1. **Durable amendment history (Required Change 1)**: Made `amendment_queued` journal persistence part of the success path for both `add_manual_amendment` and `stage_amendment_batch`. Previously, journal append was best-effort (`let _ =`), meaning a successful command could leave no journal event. Now:
   - `add_manual_amendment`: saves the pre-mutation snapshot before any changes. After the snapshot is committed, the pre-built journal line is durably appended. If the append fails, the snapshot is restored to pre-mutation state and the amendment file is removed, then the journal error is returned.
   - `stage_amendment_batch`: saves the pre-mutation snapshot. After the snapshot is committed, all journal events are pre-serialized (serialization failures also trigger rollback). Each event is then durably appended. If any append fails, the old snapshot is restored and all staged amendment files are removed.

2. **Test coverage (Recommended Improvement)**: Added `FailingAppendJournalStore` fixture (journal read succeeds, append always fails) and two new unit tests:
   - `add_manual_amendment_fails_when_journal_append_fails` â€” verifies amendment file is rolled back and snapshot is restored when journal append fails.
   - `stage_amendment_batch_fails_when_journal_append_fails` â€” verifies all staged amendment files are rolled back and snapshot is restored when journal append fails during batch staging.

3. **Conformance coverage**: Added `parity_slice3_journal_append_failure_rollback` scenario and feature file entry. The scenario uses `RALPH_BURNING_TEST_JOURNAL_APPEND_FAIL_AFTER=0` via `run_cli_with_env` to inject journal append failure during `project amend add`, then verifies:
   - The command fails (non-zero exit)
   - No amendment appears in `project amend list`
   - `run.json` has no pending amendments

4. **Docs updated**: `amendments.md` failure safety section updated to document that journal append is durable (not best-effort) and append failures trigger full rollback. `slice-3.md` updated with iteration 6 changes.

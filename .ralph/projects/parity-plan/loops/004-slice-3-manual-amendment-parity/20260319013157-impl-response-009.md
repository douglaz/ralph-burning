# Implementation Response (Iteration 9)

## Changes Made
1. **Pre-commit amendment rollback (Required Change 1)**: `add_manual_amendment` and `stage_amendment_batch` no longer swallow cleanup failures before the journal phase. All `let _ =` patterns on file cleanup after snapshot/reopen or mid-batch write errors are replaced with explicit error checking. If file cleanup fails after the primary error, a `CorruptRecord` error is returned with both the original error and the cleanup failure details. This applies to:
   - `add_manual_amendment`: snapshot/reopen write failure + file cleanup failure at service.rs:854-867
   - `stage_amendment_batch`: mid-batch file write failure + earlier file cleanup failure at service.rs:1107-1123
   - `stage_amendment_batch`: snapshot/reopen write failure + file cleanup failure at service.rs:1151-1168
   - `stage_amendment_batch`: serialization failure + snapshot restore/file cleanup failure at service.rs:1191-1215

2. **Remove/clear failure invariants (Required Change 2)**: `remove_amendment` and `clear_amendments` no longer ignore `write_amendment` failures when restoring deleted files after a snapshot error. In all cases, if rollback cannot fully restore pre-command state, a `CorruptRecord` error is returned with both the original failure and the restore failure. This applies to:
   - `remove_amendment`: snapshot write failure + file restore failure at service.rs:964-978
   - `clear_amendments` (all deleted): snapshot write failure + file restore failure at service.rs:1031-1046
   - `clear_amendments` (partial): repair snapshot write failure + file restore failure at service.rs:1061-1076

3. **Failure-injection test coverage (Recommended Improvement)**: Added 3 new test fixtures and 5 new unit tests:
   - `AlwaysFailingSnapshotStore` ‚Äî reads succeed, every write fails
   - `FailingWriteAmendmentQueue` ‚Äî remove succeeds, write always fails
   - `PartialRemoveFailingWriteQueue` ‚Äî first N removes succeed, rest fail; write always fails
   - `add_manual_amendment_returns_corrupt_when_snapshot_and_cleanup_both_fail`
   - `stage_amendment_batch_returns_corrupt_when_snapshot_and_cleanup_both_fail`
   - `remove_amendment_returns_corrupt_when_snapshot_and_restore_both_fail`
   - `clear_amendments_returns_corrupt_when_snapshot_and_restore_both_fail`
   - `clear_amendments_partial_returns_corrupt_when_repair_and_restore_both_fail`

4. **Docs updated**: `amendments.md` failure safety section updated to document composite `CorruptRecord` errors for all rollback-failure paths. `slice-3.md` updated with iteration 9 changes.

# Implementation Response (Iteration 12)

## Changes Made
1. **`project amend clear` result ordering (Required Change 1)**: Restructured `handle_amend_clear` in `project.rs` so that `lock_guard.close()` no longer hides the partial-clear contract. Previously, `lock_guard.close()?;` on line 806 propagated `GuardCloseFailed` before the `result` match, hiding the exact removed/remaining IDs from `AmendmentClearPartial`. Now the close result is captured into `let close_result = lock_guard.close();` and deferred:
   - On successful clear (`Ok`): `close_result?` still propagates any close failure.
   - On partial clear (`AmendmentClearPartial`): removed/remaining IDs are always printed to stderr, the close failure is noted as a secondary diagnostic, and `AmendmentClearPartial` is returned as the primary error.
   - On other errors: propagated as before.

2. **Combined regression test (Recommended Improvement 1)**: Added `cli_project_amend_clear_partial_failure_surfaces_ids_despite_close_failure` CLI test that enables both `RALPH_BURNING_TEST_AMENDMENT_REMOVE_FAIL_AFTER=1` and `RALPH_BURNING_TEST_DELETE_LOCK_BEFORE_CLOSE=1` simultaneously, verifying that removed/remaining IDs and the close failure message all appear in stderr.

3. **Docs updated**: `amendments.md` lease conflict section and `slice-3.md` updated with iteration 12 changes and test counts (18 CLI tests).

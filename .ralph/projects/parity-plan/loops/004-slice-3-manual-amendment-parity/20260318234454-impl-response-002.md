# Implementation Response (Iteration 2)

1. **Amendment list source of truth (Required Change 1)**: `service::list_amendments` now reads from the canonical `RunSnapshot.amendment_queue.pending` via `RunSnapshotPort` instead of the file-backed `AmendmentQueuePort`. 2. **CLI integration tests green (Required Change 2)**: All 5 failing tests updated to match the `Amendment: <id>` CLI output format:

   - `project_amend_duplicate_manual_add_is_noop` — same parsing for `first_id`

3. **Conformance coverage (Required Change 3)**: Three sub-fixes:
   - `parity_slice3_completion_blocking`: Now asserts `interrupted_run.stage_cursor.stage` is the planning stage (proving completion was unwound), then attempts `run start` and verifies the engine either blocks with a completion-blocked error, processes the amendment before completing, or leaves the project in a non-completed state.

# Implementation Response (Iteration 1)

## Changes Made
1. **Canonical amendment state (Required Change 1)**: `add_manual_amendment`, `remove_amendment`, `clear_amendments`, and `reopen_completed_project` now update `snapshot.amendment_queue.pending` in `run.json` atomically alongside disk operations. Added `reopen_completed_project_with_snapshot()` to support callers that hold an already-modified snapshot. The `remove_amendment` and `clear_amendments` service functions now accept `RunSnapshotPort` and `RunSnapshotWritePort` to read/write the canonical snapshot.

2. **PR-review staging parity (Required Change 2)**: PR-review ingestion now routes through a new shared `stage_amendment_batch()` service function instead of writing amendments directly via `amendment_queue.write_amendment()`. The shared service handles dedup, journal persistence, canonical snapshot sync, and completed-project reopen consistently for both manual and automated paths. `PrReviewIngestionService` now takes a `JournalStorePort` parameter, and the old `stage_amendments()` / `reopen_completed_project()` methods on the service were removed in favor of the shared path.

3. **Operator-facing amendment contract (Required Change 3)**: `project amend list` now surfaces `dedup_key` (truncated to 12 chars) alongside amendment ID, source, and body preview. `project amend add` output changed from bare `{id}` to `Amendment: {id}` for consistency with conformance expectations. Partial `clear` failures now report the exact removed and remaining amendment IDs in CLI output. Body truncation uses char-aware `truncate_utf8()` instead of byte-index slicing.

4. **Conformance deliverables (Required Change 4)**: Added 4 new `parity_slice3_*` scenarios to both the `.feature` file and the executor registrations:

   - `parity_slice3_completion_blocking` â€” pending amendments block completion and populate `run.json`
   - `parity_slice3_lease_conflict_rejection` â€” writer lease held prevents amendment mutation
   - `parity_slice3_run_json_sync` â€” verifies `run.json` `amendment_queue.pending` reflects added amendments

   Executor assertions now match the `Amendment: <id>` CLI output format.

5. **Recommended: RAII writer lease**: `handle_amend_add` now acquires a `CliWriterLeaseGuard` for the duration of the mutation instead of probe-and-release.

6. **Recommended: UTF-8 safe truncation**: Added `truncate_utf8()` helper that truncates at char boundaries.

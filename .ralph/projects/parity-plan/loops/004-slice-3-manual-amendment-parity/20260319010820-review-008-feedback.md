## Required Changes
1. **Shared batch journal atomicity**: `stage_amendment_batch()` still leaves durable history behind when the second-or-later `amendment_queued` append fails.
   - Current: the shared PR-review staging path appends journal lines one by one at [service.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/project_run_record/service.rs#L1144). If a later append fails, it only tries to restore `run.json` and delete amendment files at [service.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/project_run_record/service.rs#L1146) and [service.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/project_run_record/service.rs#L1148), but any earlier journal lines are already permanent. The filesystem failpoint explicitly fails after `N` successful appends at [fs.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/adapters/fs.rs#L92) and [fs.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/adapters/fs.rs#L703), so a 2+ amendment PR-review batch can still end with `journal.ndjson` saying an amendment was queued while canonical pending state was rolled back.
   - Expected: the shared staging path needs all-or-nothing durable history for multi-amendment batches, or it must surface partial-journal persistence as an unrecovered consistency failure instead of reporting a clean rollback.
   - Reference: Feature spec bullets “Manual amendment ingestion appears in durable project history via `amendment_queued` events...” and “The shared amendment staging service is reused by PR-review ingestion...”; fixed invariant “canonical state is structured state plus journal”.

2. **Rollback failure handling**: amendment intake still swallows rollback failures after journal append errors.
   - Current: both manual add and shared batch staging ignore errors when restoring the old snapshot and removing written amendment files at [service.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/project_run_record/service.rs#L865), [service.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/project_run_record/service.rs#L866), [service.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/project_run_record/service.rs#L1146), and [service.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/project_run_record/service.rs#L1148). If rollback itself fails, the command returns only the journal error while mutated amendment state may still remain visible.
   - Expected: if rollback cannot fully restore pre-command state, return an explicit rollback failure that includes both the append error and the restore/cleanup failure, matching the stronger combined-error pattern already used elsewhere in this service at [service.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/project_run_record/service.rs#L658) and [service.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/project_run_record/service.rs#L666).
   - Reference: Feature spec bullet “Failure invariants are explicit and tested...” and fixed invariant “canonical state is structured state plus journal”.

# Implementation Response (Iteration 4)

## Changes Made
1. **Required #1 — Answer-boundary rollback**: Fixed `answer()` in `service.rs` to snapshot `question_round`, `status`, and `status_summary` before advancing past the answer boundary. The run remains pinned to the pre-answer question state and can be retried.

2. **Required #2 — Question-round open rollback**: Fixed `open_question_round()` in `service.rs` to snapshot `committed_stages` and `latest_question_set_id` before invalidation. On `QuestionRoundOpened` journal append failure, both fields are restored from the snapshot, and the non-durable question-set payload/artifact pair is removed before failing. The run reflects only the last durable validation boundary.

3. **Required #3 — Full-mode stage rollback**: Fixed `commit_full_mode_stage()` in `service.rs` to properly restore `current_stage` and `recommended_flow` on `StageCompleted` journal failure. `current_stage` is now derived from committed stages in pipeline order (not from `run.current_stage`, which the pipeline code advances before calling `commit_full_mode_stage`). `prior_recommended_flow` is accepted as a parameter, with the synthesis block capturing it before mutation. Added 3 new tests covering all three rollback paths.

Current: [service.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/requirements_drafting/service.rs#L875) says opening a question round “invalidates synthesis and downstream”, but the function only persists the question set and moves the run to `awaiting_answers`. 2. **Seed rollback canonical state**: full-mode seed rollback can leave `run.json` claiming a committed `project_seed` that has already been rolled back.

Expected: on every seed rollback path, restore canonical state to the last successful pre-seed boundary before calling `fail_run`.
Reference: Slice 1 acceptance criterion “If any stage write, journal append, or seed write fails, canonical state remains pinned to the last fully committed stage...”.

# Implementation Notes

## Decisions Made
- Implemented the full seven-stage pipeline (`ideation → research → synthesis → implementation_spec → gap_analysis → validation → project_seed`) with each stage producing a validated JSON payload and deterministic Markdown artifact.
- Added `FullModeStage` enum with `pipeline_order()`, `downstream_stages()`, and `question_round_invalidated()` for ordered traversal and selective invalidation.
- Implemented deterministic cache-key reuse via `compute_stage_cache_key()` using `DefaultHasher` over stage name + upstream concatenation, stored in `committed_stages` BTreeMap.
- Quick-mode revision loop bounded by `MAX_QUICK_REVISIONS=5` with `RevisionFeedback` payload capturing reviewer suggestions per round.
- Versioned `ProjectSeedPayload` at v2 with `SeedSourceMetadata` (mode, committed stages, timing). v1 still supported for backward compatibility via `default_seed_version()`.
- Validation `NeedsQuestions` outcome triggers question-round opening that invalidates `synthesis` and downstream while preserving `ideation`/`research`.
- All new `RequirementsRun` fields use `#[serde(default)]` so pre-Slice-1 `run.json` files deserialize correctly.
- Shared `generate_and_commit_seed()` between full and quick modes to avoid duplication.
- Added stage-aware output to `requirements show`: current stage, completed stages, quick revision count, cache reuse indicator, pending question count, and suggested `project create` command.

## Spec Deviations
- Cache reuse conformance scenarios test the mechanism via `run.json` state inspection rather than mocking dual invocations, since the stub backend does not support stateful call counting.
- Question-round invalidation conformance scenario is specified in the feature file but not implemented as a conformance executor — would require a multi-step stub backend that returns `needs_questions` on first validation then `pass` on retry, which exceeds stub capabilities. Covered by unit tests instead.

- `cargo check` — 0 errors, 0 warnings

- 5 conformance scenario executors: full-mode staged happy path, quick-mode revision loop, versioned seed output, show stage progress, backward-compat run.json

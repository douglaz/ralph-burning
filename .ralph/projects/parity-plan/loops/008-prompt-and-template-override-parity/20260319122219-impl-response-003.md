# Implementation Response (Iteration 3)

## Changes Made
1. **Requirements project-override parity** (`service.rs`, `requirements.rs`, `project.rs`, `daemon_loop.rs`): Wired `project_id: Option<&ProjectId>` through the entire requirements service execution path. Updated public APIs (`draft()`, `quick()`, `answer()`) and all internal methods (`run_full_mode_pipeline()`, `run_quick_mode_pipeline()`, `open_question_round()`, `generate_and_commit_seed()`, `build_draft_prompt()`) to accept and forward an optional project ID. All 11 `resolve_and_render()` calls in `service.rs` now pass `project_id` instead of hardcoded `None`. CLI callers pass `None` when no project context exists. Updated `docs/templates.md` to document unified three-tier precedence for all prompt surfaces including requirements.

2. **Malformed panel override failure invariants** (`engine.rs`): Added pre-validation of all panel template overrides before `stage_entered` journal writes in all three panel dispatch functions:
   - `dispatch_prompt_review_panel()`: resolves `prompt_review_refiner` and `prompt_review_validator` before durable state
   - `dispatch_completion_panel()`: resolves `completion_panel_completer` before durable state
   - `dispatch_final_review_panel()`: resolves `final_review_reviewer`, `final_review_voter`, and `final_review_arbiter` before durable state

   A malformed panel override now causes failure before any `stage_entered` event or snapshot mutation.

3. **Unknown placeholder rejection is incomplete** (`template_catalog.rs`): Added `extract_all_marker_tokens()` that captures every `{{...}}` token regardless of character validity (including hyphens, spaces, etc.). Updated `validate_template()` to compare all marker tokens against valid placeholders — tokens with invalid characters are rejected as malformed overrides. Updated the internal test that previously codified the ignore behavior. Added 2 unit tests (`placeholder_with_hyphens_rejected_as_malformed`, `placeholder_with_spaces_rejected_as_malformed`) and 1 conformance scenario (`@parity_slice7_invalid_marker_rejection`).

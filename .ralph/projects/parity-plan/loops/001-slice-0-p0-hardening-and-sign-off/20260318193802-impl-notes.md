# Implementation Notes

## Decisions Made
- Hardened `ProcessBackendAdapter::check_availability()` to require an executable file on `PATH` and to return `BackendUnavailable` with the exact non-executable path when a binary is found but cannot be executed.
- Threaded a single resolved final-review planner target through backend policy resolution, final-review execution, snapshot persistence, and resume drift comparison so planner drift is detected independently of reviewer or arbiter drift.
- Added `parity_slice0_*` conformance coverage for executable permission checks, cancel/timeout cleanup, final-review planner snapshotting and drift detection, and GitHub compare-ref encoding.
- Added the Slice 0 report at [`docs/slice-reports/slice-0.md`](/root/new-ralph-burning/ralph-burning-rewrite/docs/slice-reports/slice-0.md) and recorded this loop’s notes at [`20260318193709-impl-notes.md`](/root/new-ralph-burning/ralph-burning-rewrite/20260318193709-impl-notes.md).

## Spec Deviations
- Used `nix::sys::signal::kill` instead of a direct `libc::kill` call. The crate globally forbids `unsafe` code, so a literal `libc::kill` implementation would not compile; the safe `nix` wrapper preserves the required in-process POSIX signal behavior and `ESRCH` handling.

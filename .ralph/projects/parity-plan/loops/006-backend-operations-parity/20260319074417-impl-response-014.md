# Implementation Response (Iteration 14)

## Changes Made
1. **`backend show-effective` now correctly distinguishes `settings.default_model` from models embedded in `default_backend`**: Fixed `show_effective()` in `diagnostics.rs` to compare `bp.default_model` with `bp.base_backend.model` — when they match, the model came from `settings.default_model` and its source is reported via `source_for_default_model()` (e.g. `"workspace.toml"`); when they differ, the model was embedded in `default_backend` and is reported via `source_for_base_backend()`. The same fix was applied to `model_source_for_role()` so inherited roles also report the correct source layer. Previously, `[settings] default_model = "my-model"` in workspace.toml caused both the top-level `default_model.source` and inherited roles' `model_source` to report `"default"` instead of `"workspace.toml"`.

2. **`backend list --json` `compile_only` field now always serialized**: Removed `#[serde(skip_serializing_if = "Option::is_none")]` from `BackendListEntry.compile_only`, so the field is always present in JSON output. Non-compile-only backends now emit `"compile_only": null` instead of omitting the field, matching the documented schema (`"bool | null"`). Updated conformance scenario comment to reflect the field is present as `null` rather than absent.

3. **CLI failure-path coverage expanded** with 4 new tests (19 total backend CLI tests):
   - `backend_list_nonzero_exit_without_workspace` — no workspace initialized, asserts non-zero exit
   - `backend_show_effective_nonzero_exit_without_workspace` — no workspace initialized, asserts non-zero exit
   - `backend_list_nonzero_exit_with_corrupt_config` — corrupt workspace.toml, asserts non-zero exit
   - `backend_show_effective_nonzero_exit_with_invalid_override` — invalid `--backend` value, asserts non-zero exit

4. **New unit tests** (2 added, 68 total backend diagnostics unit tests):
   - `show_effective_default_model_from_settings_default_model_reports_correct_source` — sets only `settings.default_model`, verifies top-level `default_model.source` traces to workspace.toml and inherited planner `model_source` is also correct
   - `show_effective_default_model_embedded_in_default_backend_beats_settings_default_model` — sets both embedded model and separate `default_model`, verifies embedded model takes precedence

5. **Docs updated**: `cli-reference.md` clarifies `compile_only` is always present in JSON and documents `default_model` vs `default_backend` source distinction. `slice-5.md` test counts updated to 68 unit / 19 CLI tests with gap notes for all three fixes.

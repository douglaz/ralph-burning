# Review: SUGGESTIONS

## Required Changes
1. **Panel probe timeout fidelity**: `backend probe` does not always report the same timeout that execution would use for the target it displays.
   - Current: [diagnostics.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/diagnostics.rs#L604) uses `BackendPolicyRole::FinalReviewer` for `final_review_panel.target.timeout_seconds`, while runtime uses planner timeout at [engine.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L6340). [diagnostics.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/diagnostics.rs#L655) likewise uses `PromptValidator` for `prompt_review_panel.target`, while runtime uses `PromptReviewer` at [engine.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L5878). Direct repro: with `[backends.claude.role_timeouts] planner=11 final_reviewer=22`, `backend probe --role final_review_panel --flow standard --json` returns `target.timeout_seconds = 22`.
   - Expected: Compute `target.timeout_seconds` from the actual target role being exposed in the DTO, or change the DTO so it no longer claims planner/refiner timeout semantics it does not honor.
   - Reference: Slice 5 acceptance bullet that `backend probe --role <role> --flow <flow> [--cycle <n>]` uses the same resolution paths as run execution.

2. **Probe failure detail contract**: required probe failures still do not identify the effective config source field, and the primary target error is only labeled as `planner/primary`.
   - Current: [diagnostics.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/diagnostics.rs#L694) says probe failures include exact member identity and source field, but the actual error paths at [diagnostics.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/diagnostics.rs#L709), [diagnostics.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/diagnostics.rs#L725), and [diagnostics.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/diagnostics.rs#L750) only return plain `BackendUnavailable` text with no `workflow.planner_backend`, `final_review.arbiter_backend`, `final_review.backends`, etc. Direct repro: `PATH=/tmp ... backend probe --role final_review_panel --flow standard --json` exits with `backend 'claude' is unavailable: required target 'final_review_panel' (planner/primary) unavailable...` and no config-source detail.
   - Expected: Required probe failures should report the exact panel member/role and the effective source field that selected it, matching the spec’s operator-diagnostics requirement for `probe` as well as `check`.
   - Reference: Slice 5 acceptance bullet that required panel failures identify the exact role/panel member, backend family, and effective config source field for `probe` or `check`.

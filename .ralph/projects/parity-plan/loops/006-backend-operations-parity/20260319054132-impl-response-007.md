1. **`backend check` optional-member readiness semantics**: Rewrote `check_backends_with_availability()` to separate required targets from optional panel members. Added `check_panel_availability()` helper that:

   - Required unavailable members → `AvailabilityFailure` with exact member identity
   - Optional unavailable members → silently omitted (not reported as failures)
   - After all members checked, if available count < minimum → `PanelMinimumViolation`
   - Stage-role targets, planners, arbiters, and refiners remain always-required

2. **`backend show-effective` surfaces broken role resolution**: Changed `filter_map` → `map` with error handling. When `resolve_role_target` fails, the role is still included in the output with:

   - `model_id`, `model_source`, `timeout_source` set to `"unresolved"`
   - `resolution_error` set to the error message
   - `override_source` still reflects the workspace.toml configuration
   - Added `resolution_error: Option<String>` field to `RoleEffectiveView` DTO (skipped in JSON when `None`)
   - Updated text renderer to show `[UNRESOLVED: <error>]` for broken roles

3. **`backend probe` failure identity**: Fixed both config-time and availability-time probe failures:
   - Config-time: `self.probe()` errors are now wrapped via `.map_err()` to include exact target identity (`probe_primary_target_label`) and source (`probe_primary_config_source`)
   - Availability-time: Replaced hard-coded `(planner/primary)` with dynamic `(planner)`, `(refiner)`, or the singular role name via `probe_primary_target_label()`
   - Added `probe_primary_target_label()` — returns `"planner"` for completion/final-review panels, `"refiner"` for prompt-review, or the role itself for singular probes
   - Added `probe_primary_family()` — returns the configured backend family for the primary target, used to annotate config-time failures
   - Added `panel_member_label_prefix()` — returns qualified member labels (`completion_panel.member`, `final_review_panel.reviewer`, `prompt_review_panel.validator`) for panel member availability failures

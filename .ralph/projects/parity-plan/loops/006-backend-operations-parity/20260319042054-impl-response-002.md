# Implementation Response (Iteration 2)

## Changes Made
1. **Active project flow scoping**: `handle_check` now calls `resolve_active_project_flow()` which reads the canonical `ProjectRecord::flow` from the active project via `FsProjectStore`. Falls back to `config.default_flow()` only when no project is selected or the record can't be read.

2. **Availability coverage and failure identity**: `check_backends_with_availability()` no longer short-circuits on config failures. It now walks panel-specific targets (completion members, final-review reviewers + arbiter, prompt-review validators + refiner) in addition to stage-role targets. Availability failures report the actual role/panel member identity (e.g. `"final_review_panel.arbiter"`) instead of generic `"availability"`.

3. **Optional unavailable panel members**: Added `probe_with_availability()` async method. When an adapter is available, the CLI uses this path; optional members that are enabled but fail availability are moved to `omitted`. `build_panel_member_views` also avoids double-counting already-resolved members.

4. **`show-effective` source precedence and docs contract**: `override_source_for_role` now reports the actual source layer for inherited roles as `"default_backend (workspace.toml)"` instead of bare `"default"`. Docs updated: added `session_policy` per role, `availability_failure` failure kind, `arbiter` in probe panel schema, renamed top-level `session_policy` → `default_session_policy`.

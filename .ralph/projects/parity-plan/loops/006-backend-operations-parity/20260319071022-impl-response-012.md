# Implementation Response (Iteration 12)

## Changes Made
1. **Panel member identity now uses original configured-spec index**: Added `configured_index: usize` to `ResolvedPanelMember` in `policy.rs`, tracking each member's position in the original configured spec list through `resolve_panel_backends()`. Updated all downstream consumers:
   - `check_panel_availability()` now receives `(target, required, configured_index)` tuples and uses `configured_index` in failure role labels instead of the post-filtering enumerate index
   - `PanelMemberView` DTO carries `configured_index` (serialized in JSON output)
   - `build_panel_member_views()` propagates configured index from resolved members to view DTOs
   - `probe_with_availability()` uses `member.configured_index` instead of the drain enumerate index when reporting required-member availability failures
   - All `ResolvedPanelMember` construction sites updated (policy.rs, final_review.rs, scenarios.rs, workflow_engine_test.rs)

   Before: with `["?openrouter", "claude"]` and disabled openrouter, a failing required claude at spec[1] was reported as `member[0]`. After: correctly reports `member[1]`.

2. **`backend list` `compile_only` for stub is now build-sensitive**: Changed the hard-coded `compile_only = Some(true)` for `BackendFamily::Stub` in `list_backends()` to be `#[cfg]`-gated: `None` when `test-stub` feature is active (stub is operational), `Some(true)` when absent (stub is compile-time-only). Updated the existing `list_backends_returns_all_families` test assertion to be feature-conditional.

   - `list_backends_stub_not_compile_only_in_stub_build` (cfg test-stub) — verifies `compile_only` is `None` for stub in test-stub builds

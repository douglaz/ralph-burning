# Implementation Response (Iteration 1)
## Changes Made
1. **Flow-scoped diagnostics**: `check_backends` now resolves the `FlowDefinition` and scopes all checks to the flow's actual stages. `probe` validates that the requested panel target exists in the given flow, returning a clear error when it does not. `docs_change` and `ci_improvement` flows correctly skip completion/final-review checks.

2. **`show-effective` contract**: Each `RoleEffectiveView` now includes a `session_policy` field matching runtime behavior — `"new_session"` for panel roles, `"reuse_if_allowed"` for main stage roles. Source precedence for `PromptReviewer` and `Arbiter` roles now resolves via the actual config keys instead of falling through to `"default"`.

3. **Panel probe and failure detail**: `probe_final_review_panel` now serializes the arbiter as a dedicated field. Panel check failures report the exact backend family and config field path instead of generic `"mixed"` / `"*_panel config"`.

4. **Readiness validation**: Added `check_backends_with_availability()` that calls `adapter.check_availability()` for each unique resolved target via the `AgentExecutionPort` trait. The CLI `backend check` handler now uses this path with the production adapter.

## Required Changes
1. **`backend show-effective` misattributes models that come from `settings.default_model`**: the JSON/text source-precedence view is still wrong when the model is configured separately from `default_backend`.
Current: [config.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workspace_governance/config.rs#L364) merges `default_model` into `base_backend.model`, but [diagnostics.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/diagnostics.rs#L664) and [diagnostics.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/diagnostics.rs#L1392) then treat any populated `base_backend.model` as if it came from `default_backend`. Direct repro on `HEAD`: with only `[settings] default_model = "my-default-model"` in `workspace.toml`, `backend show-effective --json` reports `default_model.source = "default"` and inherited roles like `planner` report `model_source = "default"` even though the model is coming from workspace config.
Expected: distinguish an embedded model in `default_backend = "family(model)"` from a separately configured `default_model`, and report the actual source layer for both the top-level `default_model` field and inherited roles’ `model_source`.

# Implementation Response (Iteration 6)

## Changes Made
1. **Exact panel-failure identity in `backend check`**: Replaced the generic `check_panel_members()` dispatch with three decomposed methods — `check_completion_panel_config()`, `check_final_review_panel_config()`, and `check_prompt_review_panel_config()` — that check each component (arbiter, refiner, individual reviewers/validators) separately and report the exact failing member and its selecting config field:
   - `final_review_panel.arbiter` with source `final_review.arbiter_backend`
   - `prompt_review_panel.refiner` with source `prompt_review.refiner_backend`
   - `final_review_panel.reviewer[N]` with source `final_review.backends`
   - `completion_panel.member[N]` with source `completion.backends`
   - `prompt_review_panel.validator[N]` with source `prompt_review.validator_backends`
   - Removed the now-unused `classify_panel_error_structured` helper.

2. **Per-field source precedence in `backend show-effective`**: Added `model_source` and `timeout_source` fields to `RoleEffectiveView` DTO. The `show_effective()` method now traces the resolution origin of each role's model_id (through explicit selection model → role_models config → default_model → family default) and timeout_seconds (through role_timeouts config → backend timeout_seconds → global default), reporting the exact config layer source for each.

3. **New tests** (5 added, 44 total backend diagnostics tests):
   - `check_arbiter_failure_reports_exact_member_and_source` — disables openrouter, configures arbiter to use it, asserts failure role is `final_review_panel.arbiter` with source `final_review.arbiter_backend`.
   - `check_refiner_failure_reports_exact_member_and_source` — disables openrouter, configures refiner to use it, asserts failure role is `prompt_review_panel.refiner` with source `prompt_review.refiner_backend`.
   - `show_effective_reports_model_source_from_role_models` — sets `backends.claude.role_models.planner`, asserts `model_source` is `workspace.toml`.
   - `show_effective_reports_timeout_source_from_role_timeouts` — sets `backends.claude.role_timeouts.planner`, asserts `timeout_source` is `workspace.toml`.
   - `show_effective_default_model_and_timeout_source` — with no explicit role config, asserts both `model_source` and `timeout_source` are `default`.

4. **Updated existing tests**: `check_reports_required_panel_member_failure` and `check_panel_failure_identifies_exact_backend_family` now look for `completion_panel.member[N]` instead of the old collapsed `completion_panel` role identity.

5. **Docs updated**: `cli-reference.md` — added `model_source`/`timeout_source` to show-effective JSON schema, documented per-member failure identity for check. `slice-5.md` — updated test count to 44 and documented new behaviors.

# Review: SUGGESTIONS

## Required Changes
1. **`backend check` optional-member readiness semantics**: optional panel members are still treated as blocking failures during live availability checks.
Current: [diagnostics.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/diagnostics.rs#L317) queues every resolved panel member for availability checks, and [diagnostics.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/diagnostics.rs#L377) turns any availability error into a failure without consulting `required`. Direct repro: with `[completion] backends = ["claude", "codex", "?openrouter"]`, `min_completers = 2`, dummy `claude`/`codex` executables on `PATH`, and `OPENROUTER_API_KEY` unset, `backend check --json` exits non-zero on `completion_panel.member[2]` even though the two required members satisfy the minimum.
Expected: optional unavailable members must be omitted from readiness evaluation and only fail the command if their omission drops the panel below its configured minimum, in which case the failure should be a minimum violation rather than a generic availability failure.
Reference: Slice 5 effective-required and optional-member acceptance bullets: [20260319032842-spec.md#L26](/root/new-ralph-burning/.ralph/projects/parity-plan/loops/006-backend-operations-parity/20260319032842-spec.md#L26), [20260319032842-spec.md#L30](/root/new-ralph-burning/.ralph/projects/parity-plan/loops/006-backend-operations-parity/20260319032842-spec.md#L30)

2. **`backend show-effective` hides broken role resolution**: the command silently drops roles whose configured backend cannot currently resolve.
Current: [diagnostics.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/diagnostics.rs#L565) uses `filter_map`, so any `resolve_role_target` failure removes that role from the output entirely. Direct repro: with `[workflow] planner_backend = "openrouter"` and `[backends.openrouter] enabled = false`, `backend show-effective --json` succeeds but omits the `planner` entry from `roles`.
Expected: `show-effective` must surface every role's effective selection/source information, including broken selections, or fail explicitly; it cannot silently hide unresolved roles in a command whose purpose is pre-run diagnostics.
Reference: Slice 5 `show-effective` acceptance and done-when clauses: [20260319032842-spec.md#L28](/root/new-ralph-burning/.ralph/projects/parity-plan/loops/006-backend-operations-parity/20260319032842-spec.md#L28), [prompt.md#L242](/root/new-ralph-burning/.ralph/projects/parity-plan/prompt.md#L242)

3. **`backend probe` failure identity is still incomplete/misleading**: probe failures do not consistently report the exact failing target/member and source field.
Current: [diagnostics.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/diagnostics.rs#L799) lets config-time probe failures bubble out of `self.probe(...)` unchanged, so a disabled required completion member currently returns only `backend 'openrouter' is unavailable: required backend is disabled or unavailable` with no member identity or source. Separately, [diagnostics.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/diagnostics.rs#L812) hard-codes `(planner/primary)` for primary-target availability failures, so `backend probe --role prompt_review_panel --flow standard` reports the refiner failure as `planner/primary`.
Expected: both config-time and availability-time probe failures should identify the exact failing target (`completion_panel.member[n]`, `final_review_panel.reviewer[n]`, `prompt_review_panel.refiner`, `reviewer`, etc.), the backend family, and the selecting config field.
Reference: Slice 5 probe and exact-failure acceptance bullets: [20260319032842-spec.md#L29](/root/new-ralph-burning/.ralph/projects/parity-plan/loops/006-backend-operations-parity/20260319032842-spec.md#L29), [20260319032842-spec.md#L31](/root/new-ralph-burning/.ralph/projects/parity-plan/loops/006-backend-operations-parity/20260319032842-spec.md#L31)

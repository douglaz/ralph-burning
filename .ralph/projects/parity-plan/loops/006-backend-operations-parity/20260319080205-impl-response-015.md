# Implementation Response (Iteration 15)

## Changes Made
1. **`backend check` now uses the runtime completion-panel resolution path when `completion.backends` is implicit**: Fixed `check_completion_panel_config()` in `diagnostics.rs` to check `config.completion_backends_are_explicit()` first. When implicit, the check validates via `resolve_role_target(BackendPolicyRole::Completer, 1)` — the same path that `default_completion_targets()` uses at runtime — instead of iterating the built-in default backend list. This prevents false failures on default-list backends that runtime would never use.

2. **`backend check` now validates final review based on flow stage plan, not `final_review.enabled`**: Removed the `self.config.final_review_policy().enabled` gate from both `check_backends()` and `check_backends_with_availability()`. The engine's `stage_plan_for_flow()` does NOT filter `FinalReview` based on that flag, so diagnostics now matches.

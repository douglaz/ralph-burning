# Review: SUGGESTIONS

## Required Changes
1. **Flow-scoped diagnostics**: `backend check` and `backend probe` do not honor the active project flow or the `--flow` argument, so they can report panel requirements that the selected flow never executes.
   - Current: `handle_check` always passes `config.default_flow()` and `BackendDiagnosticsService::check_backends` ignores the parameter entirely; `probe` just echoes `flow` while resolving the same targets for every flow. Direct repro: `ralph-burning backend probe --role completion_panel --flow docs_change --json` succeeds even though `docs_change` has no `completion_panel`. Code: [backend.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/cli/backend.rs#L182), [backend.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/cli/backend.rs#L240), [diagnostics.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/diagnostics.rs#L186), [diagnostics.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/diagnostics.rs#L336), [mod.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/mod.rs#L52), [mod.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/mod.rs#L59)
   - Expected: Scope `check` to the active project's actual flow when a project is selected, and make `probe --flow` validate and resolve only the stages and panels present in that flow using the same flow-aware paths as run execution.
   - Reference: Feature spec acceptance bullets for "active project/workspace scope" and "`backend probe --role <role> --flow <flow>` uses the same resolution paths as run execution"

2. **`show-effective` contract**: the current output is missing the required session-policy detail and reports incorrect source precedence for inherited and panel-specific roles.
   - Current: `show_effective` returns one hard-coded `"session_policy": "reuse_if_allowed"` for the whole view, and `override_source_for_role` only handles planner/implementer/reviewer/qa. Roles inheriting from `default_backend` or using `prompt_review.refiner_backend` / `final_review.arbiter_backend` are reported as `"default"` even when workspace/project/CLI config selected them. Code: [diagnostics.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/diagnostics.rs#L309), [diagnostics.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/diagnostics.rs#L325), [diagnostics.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/diagnostics.rs#L585). Actual execution uses `ReuseIfAllowed` for ordinary stages but `NewSession` for completion/prompt-review/final-review panel work: [engine.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L4600), [completion.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/completion.rs#L266), [prompt_review.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/prompt_review.rs#L299), [final_review.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/final_review.rs#L814)
   - Expected: Report session policy at the same granularity the runtime uses, and compute source precedence from the actual resolved field path, including inheritance from `default_backend` and the prompt/final-review-specific override fields.
   - Reference: Feature spec acceptance bullet for "`backend show-effective` exposes ... session reuse policy, and per-field source precedence across default, workspace, project, and CLI override layers"

3. **Panel probe and failure detail**: final-review probing and panel failure reporting are still missing required operator detail.
   - Current: `probe_final_review_panel` resolves `resolution.arbiter` but never serializes it, so `backend probe --role final_review_panel --json` only exposes the planner target and reviewer members. Separately, all panel failures are emitted with `backend_family: "mixed"` and generic `config_source` strings like `"final_review config"`, leaving the exact member/source only buried in free-form text.

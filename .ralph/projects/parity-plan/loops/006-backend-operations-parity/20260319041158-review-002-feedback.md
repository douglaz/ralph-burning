Current: [`handle_check`](/root/new-ralph-burning/ralph-burning-rewrite/src/cli/backend.rs#L182) uses [`EffectiveConfig::default_flow`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workspace_governance/config.rs#L533), but the fixed project flow lives in [`ProjectRecord::flow`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/project_run_record/model.rs#L7). Direct repro: an active `docs_change` project with broken completion/final-review config still makes `backend check --json` fail on `completion_panel` and `final_review_panel`.
Expected: Resolve the active project record and scope checks to that canonical flow; only fall back to the workspace default when no project is selected.
Reference: [20260319032842-spec.md](/root/new-ralph-burning/.ralph/projects/parity-plan/loops/006-backend-operations-parity/20260319032842-spec.md#L26), [prompt.md](/root/new-ralph-burning/.ralph/projects/parity-plan/prompt.md#L228)

2. **Availability coverage and failure identity**: readiness checks still miss panel-only targets and do not preserve exact role/member identity.
Current: [`check_backends_with_availability`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/diagnostics.rs#L294) returns early on config failures, only walks stage-role targets, and serializes availability failures as `role: "availability"`. It never traverses panel-only targets from [`resolve_prompt_review_panel`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/policy.rs#L100) or [`resolve_final_review_panel`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/policy.rs#L124). Direct repro: with `[final_review] arbiter_backend = "openrouter"` and `OPENROUTER_API_KEY` unset, `backend check --json` exits `0` with `"passed": true`.
Expected: Include all effectively required resolved targets, including prompt-review refiner/validators, completion members, final-review reviewers, and arbiter; keep aggregating availability failures where resolution succeeds; report the exact role/panel member and config field instead of a generic availability bucket.
Reference: [20260319032842-spec.md](/root/new-ralph-burning/.ralph/projects/parity-plan/loops/006-backend-operations-parity/20260319032842-spec.md#L21), [20260319032842-spec.md](/root/new-ralph-burning/.ralph/projects/parity-plan/loops/006-backend-operations-parity/20260319032842-spec.md#L26), [prompt.md](/root/new-ralph-burning/.ralph/projects/parity-plan/prompt.md#L230)

1. **`backend probe` config-time panel failures still lose exact failing target identity**: panel probe config errors are still wrapped as the panel’s primary target instead of the actual failing member/arbiter.

Direct CLI repro with `[completion] backends = ["claude", "openrouter"]` plus disabled `openrouter` returns `backend 'claude' ... required target 'completion_panel' (planner) ... [source: workflow.planner_backend]` instead of the failing completion member.
   - Expected: config-time probe failures must preserve the exact failing target/member, backend family, and selecting field, e.g. `completion_panel.member[1]` with `openrouter` / `completion.backends`, and `final_review_panel.arbiter` with `openrouter` / `final_review.arbiter_backend`.
   - Reference: [diagnostics.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/diagnostics.rs#L724), [diagnostics.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/diagnostics.rs#L1007), [20260319032842-spec.md](/root/new-ralph-burning/.ralph/projects/parity-plan/loops/006-backend-operations-parity/20260319032842-spec.md#L29), [20260319032842-spec.md](/root/new-ralph-burning/.ralph/projects/parity-plan/loops/006-backend-operations-parity/20260319032842-spec.md#L31)

   - Current: [`model_source_for_role()`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/diagnostics.rs#L1235) only attributes inherited base-family models to `default_model`, but runtime resolution in [`target_for_family()`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/policy.rs#L261) actually falls through to `base_backend.model`.

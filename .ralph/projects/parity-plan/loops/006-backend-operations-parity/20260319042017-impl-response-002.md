# Implementation Response (Iteration 2)

## Changes Made
1. **Active project flow scoping**: `handle_check` now calls `resolve_active_project_flow()` which reads the canonical `ProjectRecord::flow` from the active project. Falls back to `config.default_flow()` only when no project is selected or the record can't be read. This ensures `backend check` scopes to the actual project flow, not just the workspace default.

2. **Availability coverage and failure identity**: `check_backends_with_availability()` no longer returns early on config failures — it continues to check adapter availability for all resolvable targets. Panel-specific targets are now included: completion panel members, final-review panel reviewers + arbiter, and prompt-review validators + refiner. Availability failures now report the actual role/panel member identity (e.g. `"final_review_panel.arbiter"`) instead of the generic `"availability"` string.

3. **Optional unavailable panel members**: Added `probe_with_availability()` async method on `BackendDiagnosticsService`. When an adapter is available, the CLI handler uses this path instead of the sync `probe()`. Optional panel members that are enabled but fail availability checks are moved from `members` to `omitted` with `was_optional: true`. The `build_panel_member_views` sync path also no longer double-counts members that are already in the resolved list. The CLI `handle_probe` now builds the adapter and uses availability-aware probing when possible.

4. **`show-effective` source precedence and docs contract**: `override_source_for_role` now reports the actual effective source layer for inherited roles. When a role has no explicit override and inherits from `default_backend`, the source is reported as `"default_backend (workspace.toml)"` (or whichever layer `default_backend` comes from) instead of a bare `"default"`. Roles with explicit overrides at the `Default` source level are also correctly reported as inheriting from the base. Updated `cli-reference.md`: added `session_policy` per role in the `show-effective` schema, renamed top-level `session_policy` to `default_session_policy`, added `availability_failure` to check failure kinds, added `arbiter` to probe panel schema. Updated `slice-5.md`: removed the deferred-availability gap since it's now implemented.

**Files modified:** `backend.rs`, `diagnostics.rs`, `backend_diagnostics_test.rs`, `cli-reference.md`, `slice-5.md`

**Test results:** All 27 backend diagnostics tests pass (6 new: `show_effective_inherited_role_reports_base_source`, `show_effective_explicit_role_override_beats_inheritance`, `check_with_availability_covers_panel_targets`, `check_with_availability_reports_panel_member_failure`, `probe_with_availability_omits_unavailable_optional_member`, plus existing 21).

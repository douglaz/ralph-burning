# Implementation Response (Iteration 16)

## Changes Made
1. **Opposite-family role attribution now reflects the runtime resolution path**: Rewrote `family_for_role()` in `diagnostics.rs` to mirror the resolution logic in `BackendPolicyService::resolve_role_target()`. Roles with explicit overrides still report the override's family. Roles that use planner-family resolution (Planner, Reviewer, PromptReviewer, PromptValidator, FinalReviewer, Arbiter) report `base_backend.family`. Opposite-family roles (Implementer, Qa, AcceptanceQa, Completer) now call `opposite_family(planner_family)` to determine the attempted family; when no opposite family is enabled, they report `opposite_of(<base>)` instead of incorrectly claiming the base backend. This fix affects `show-effective` (broken role `backend_family` field), `check` (failure `backend_family` field), and `probe` (error identity in `make_probe_target_error`). Added helper `role_uses_opposite_family()` to classify roles by their resolution path.

2. **Implicit completion-panel availability failures now report the correct config source**: Added `completion_panel_config_source()` helper that returns `"completion.backends"` when backends are explicitly configured, or `config_source_for_role(Completer)` (typically `"default_backend"`) when implicit. Changed `panel_members_config_source()` from a static method to an instance method so it can check config state. Updated three call sites: `check_backends_with_availability()`, `probe_completion_panel()`, and `probe_with_availability()`. Previously, all three hard-coded `"completion.backends"` even when backends came from implicit Completer-role resolution.

3. **6 new regression tests** (78 total backend diagnostics unit tests):
   - `show_effective_opposite_family_roles_report_attempted_family_not_base`
   - `show_effective_opposite_family_roles_report_resolved_family_when_available`
   - `check_reports_opposite_family_not_base_for_failed_opposite_roles`
   - `probe_singular_opposite_role_reports_correct_family`
   - `check_with_availability_implicit_completion_reports_correct_source`
   - `probe_with_availability_implicit_completion_reports_correct_source`

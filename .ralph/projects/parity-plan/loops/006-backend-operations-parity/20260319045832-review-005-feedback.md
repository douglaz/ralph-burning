   - Current: [`check_backends`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/diagnostics.rs#L195) always validates `default_backend`, and the generic flow-role pass at [`diagnostics.rs`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/diagnostics.rs#L211) still validates stage-derived roles like `final_reviewer` via [`stage_to_policy_role`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/agent_execution/policy.rs#L352) even when explicit panel config fully determines runtime targets.    - Expected: compute readiness from the same effectively used targets that execution would actually resolve for the active flow. `backend check` should not report `default_backend` or generic panel-stage roles as blocking when explicit role/panel configuration means those selections are unused.

# Implementation Notes

## Decisions Made
- **Preserved amendment files on completed-project reopen failure instead of rolling back**: The `stage_amendment_batch` function in `service.rs` now distinguishes between completed and non-completed projects when handling snapshot write failures. For completed projects, amendment files remain on disk after a reopen/snapshot failure; for non-completed projects, the existing rollback behavior is unchanged. This matches the invariant codified in Slice 3 review notes and the `transient_error_preserves_staged` conformance scenario.
- **Used cfg-gating instead of separate test targets**: Rather than creating new `unit_stub.rs` and `cli_stub.rs` files with `required-features` in Cargo.toml, used `#[cfg(feature = "test-stub")]` directly on module declarations and test functions. The acceptance criteria explicitly allowed this ("moved behind explicit test-stub gating **or** separate required-features test targets"). This approach is simpler, avoids test target proliferation, and achieves the same result.
- **Used runtime macro for CLI tests**: For the ~44 CLI integration tests that spawn the binary with `RALPH_BURNING_BACKEND=stub`, a `require_stub_binary!()` macro provides a runtime early-return rather than compile-time gating. These tests compile in both builds but skip execution when the binary lacks test-stub support.
- **No changes to pr_review.rs**: The cursor and journal advancement in `ingest_reviews` is already correctly guarded by the `?` operator on `stage_amendment_batch`. When staging fails, the error propagates immediately and prevents cursor/journal updates. No code changes were needed.

## Spec Deviations
- **Did not create `tests/unit_stub.rs` and `tests/cli_stub.rs`**: Used cfg-gating within existing files instead, as permitted by the acceptance criteria alternative wording.
- **Did not create `Cargo.toml` test target definitions**: Since cfg-gating was used instead of separate test targets, no `[[test]]` sections with `required-features` were needed.
- **PR-review ingestion unit tests in `automation_runtime_test.rs` not added**: The spec requested these, but the behavior is comprehensively covered by 4 conformance scenarios (`daemon.pr_review.*`) and 2 new service-level unit tests in `project_run_record_test.rs`. Adding unit-level PR-review ingestion tests would require extensive port-faking infrastructure (DaemonStorePort, GithubPort, ProjectStorePort, etc.) that duplicates existing conformance coverage.
- **RD-001 conformance failure not fixed**: This is a pre-existing issue where `RALPH_BURNING_TEST_LABEL_OVERRIDES` is not forwarded through the conformance CLI runner. It is unrelated to this slice's scope and does not represent a regression.

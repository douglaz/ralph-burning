## Required Changes
1. **Atomic stage-commit cleanup**: artifact-write failures can still leak durable payload history for an uncommitted stage.
   - Current: [engine.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L394) converts any `write_payload_artifact_pair` error into `fail_run`, but [fs.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/adapters/fs.rs#L676) has an explicit cleanup-failure branch that returns an error saying a payload was leaked. There is no second cleanup attempt before the run is failed, so an orphan payload file can remain under `history/payloads/`. [queries.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/project_run_record/queries.rs#L104) then treats that orphaned durable record as corruption.
   - Expected: after payload write succeeds but artifact persistence fails, the stage must leave no durable payload/artifact history behind. Either make the pair write truly atomic (for example, stage elsewhere and rename into history only on success) or complete orphan cleanup before failing the run so only raw diagnostics remain.
   - Reference: Loop 6 acceptance criteria “A stage is successful only after...” and “Failure invariants are explicit...”

## Recommended Improvements

# Implementation Response (Iteration 4)

## Changes Made
1. **Payload/artifact write failure now calls `fail_run`** (`engine.rs`): Replaced the `?` operator on `write_payload_artifact_pair` with an explicit `if let Err` block that routes through `fail_run`. After `stage_entered` is durably recorded, any payload/artifact write failure now persists a `failed` snapshot and appends a `run_failed` journal event instead of leaving the run in `running` state.

2. **Rollback failures are propagated instead of swallowed** (`engine.rs`): Both rollback sites (snapshot-write failure and journal-append failure paths) now check the result of `remove_payload_artifact_pair`. If rollback fails, the error detail includes the cleanup failure message with explicit "leaked durable history may exist" language, giving the caller visibility into the incomplete cleanup.

3. **`write_payload_artifact_pair` propagates cleanup failure** (`fs.rs`): When artifact write fails and the payload cleanup also fails, the error now includes both the original artifact write error and the cleanup failure details with the leaked payload path, instead of silently swallowing the cleanup error.

4. **`remove_payload_artifact_pair` propagates removal errors** (`fs.rs`): Changed from `let _ = fs::remove_file(...)` (silent swallow) to error collection with propagation. `NotFound` errors are still ignored (file already cleaned up), but genuine removal failures (permission errors, path-is-a-directory, etc.) are collected and returned as a compound error.

5. **`FailingPayloadArtifactWriteStore` test seam** (`workflow_engine_test.rs`): Added a new failing port that delegates to `FsPayloadArtifactWriteStore` but fails `write_payload_artifact_pair` on the Nth call. Added `payload_artifact_write_failure_persists_failed_state` test that verifies: run ends in `failed` state, no payloads/artifacts exist, no `stage_completed` events, and a `run_failed` event is present.

6. **Adapter-level rollback failure tests** (`adapter_contract_test.rs`): Added 5 new tests: `payload_artifact_write_pair_round_trip`, `payload_artifact_remove_pair_removes_both_files`, `payload_artifact_remove_pair_not_found_is_ok`, `payload_artifact_remove_pair_propagates_removal_error` (using directory-as-file trick for root compatibility), and `payload_artifact_write_pair_cleans_up_on_artifact_failure`.

# Review: SUGGESTIONS

## Required Changes
1. **Durable start/stage boundaries**: The start boundary and stage-commit boundary do not meet the spec’s atomicity requirements.
   - Current: [`engine.rs`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L173) appends `run_started` before writing `run.json`, and [`engine.rs`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L326) writes the payload/artifact pair before the `stage_completed` journal entry and next snapshot update. If journal or snapshot persistence fails at [`engine.rs`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L347) or [`engine.rs`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L365), the code returns `StageCommitFailed` without rollback or `run_failed`, so partial durable history can remain visible while `run.json` is left in the prior `running` state.
   - Expected: `run start` should persist the running snapshot before `run_started`/`stage_entered`, and a successful stage should only become visible once payload, artifact, `stage_completed`, and the next-stage snapshot are all durably committed. On journal/snapshot failure, the stage must remain uncommitted with no visible payload/artifact pair or `stage_completed`, and the run must persist failed state at the current durable boundary.
   - Reference: Loop 6 acceptance criteria “On successful start…”, “A stage is successful only after…”, and “Failure invariants are explicit…”

2. **Final review routing**: `final_review` is mapped to the wrong role, so the engine resolves the wrong backend/model for that stage.
   - Current: [`domain.rs`](/root/new-ralph-burning/ralph-burning-rewrite/src/shared/domain.rs#L237) routes `StageId::FinalReview` to `Reviewer`, and [`workflow_engine_test.rs`](/root/new-ralph-burning/ralph-burning-rewrite/tests/unit/workflow_engine_test.rs#L82) asserts that behavior.
   - Expected: `completion_panel` and `final_review` must both map to `CompletionJudge`, per the spec’s centralized stage-to-role mapping.
   - Reference: Loop 6 acceptance criterion “Stage-to-role mapping is deterministic and centralized…”

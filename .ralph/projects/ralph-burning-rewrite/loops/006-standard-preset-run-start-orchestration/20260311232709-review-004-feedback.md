# Review: SUGGESTIONS

## Required Changes
1. **Stage-commit payload/artifact write failures leave the run in `running` state**
Current: After `stage_entered` is durably recorded at [engine.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L221) and the running snapshot is rewritten at [engine.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L254), the payload/artifact write at [engine.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/engine.rs#L394) maps errors straight to `StageCommitFailed` and returns without calling `fail_run`. I reproduced this against a disposable workspace by forcing `history/payloads` to be a file: `run start` exited non-zero, `run.json` remained `status: "running"` with `active_run` still on `prompt_review`, and the journal stopped after `stage_entered`.
Expected: Any failure after `stage_entered`, including payload/artifact persistence failure, must persist a failed run boundary (`status=failed`, `active_run=null`, terminal failure recorded) and halt later stages.
Reference: Loop 6 acceptance criteria “A stage is successful only after...” and “Failure invariants are explicit...”

2. **Payload/artifact rollback is still best-effort, so uncommitted history can leak**
Current: [fs.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/adapters/fs.rs#L675) best-effort deletes the payload if artifact creation fails, and [fs.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/adapters/fs.rs#L683) / [fs.rs](/root/new-ralph-burning/ralph-burning-rewrite/src/adapters/fs.rs#L699) swallow rollback deletion errors entirely. That means the engine cannot actually guarantee “no paired history record is visible” when cleanup itself fails.
Expected: Make payload/artifact persistence truly atomic or propagate rollback failures so the engine can prevent or explicitly surface leaked durable history instead of silently continuing.
Reference: Loop 6 acceptance criteria “Persist both payload and rendered artifact atomically...” and “the stage remains uncommitted: no paired history record is visible...”

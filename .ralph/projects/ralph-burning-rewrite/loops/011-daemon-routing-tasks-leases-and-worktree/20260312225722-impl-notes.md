## Spec Deviations
- `daemon abort` immediately cleans up claimed tasks, but active-task cleanup is deferred to the running daemon loop after cancellation so the in-use worktree is not torn down mid-dispatch.
- Daemon dispatch switches the process working directory into the claimed worktree before invoking the workflow engine; engine-owned runtime/session persistence still uses the canonical project store paths.

# Implementation Notes

## Decisions Made
- Bootstrapped `ralph-burning-rewrite/` as a Rust 2021 crate named `ralph-burning` with an async `tokio` entrypoint, tracing initialization, and `clap` subcommand routing for the v1 CLI surface.
- Centralized the v1 domain model in shared code: `FlowPreset`, `StageId`, `StageCursor`, `ProjectId`, `RunId`, and `WorkspaceConfig`, with explicit cursor invariants and workspace versioning.
- Defined the four built-in flow presets as static registry entries in `workflow_composition`, so `flow list` and `flow show` share one source of truth for stage order, descriptions, and validation-profile defaults.
- Implemented workspace creation through `workspace_governance` and an `adapters::fs` filesystem adapter, creating `.ralph-burning/workspace.toml` plus the required directory stubs and rejecting implicit re-initialization.
- Added both unit and CLI integration tests so the core slice is verified at the domain/service layer and at the user-facing command layer.

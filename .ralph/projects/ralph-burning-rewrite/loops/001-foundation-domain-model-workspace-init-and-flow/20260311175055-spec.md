# Feature: Foundation — Domain Model, Workspace Init, and Flow Discovery

## Description
Bootstrap the `ralph-burning` Rust binary crate with the v1 domain model, workspace initialization, and flow discovery CLI commands. This is delivery roadmap slice 1: it establishes the Cargo project, bounded-context folder skeleton, core domain types, the `ralph-burning init` command that creates a `.ralph-burning/` workspace, and `flow list` / `flow show` commands that enumerate the four built-in flow presets with their stage sequences. It also seeds the conformance test harness with initial Gherkin scenario IDs covering workspace init and flow discovery.

## Acceptance Criteria
- [ ] `ralph-burning-rewrite/Cargo.toml` compiles a binary named `ralph-burning` (edition 2021, Rust stable)
- [ ] Dependencies include at minimum: `clap` (derive), `serde`/`serde_json`, `toml`/`toml_edit`, `thiserror`, `chrono`, and `tokio` (rt-multi-thread)
- [ ] Source tree contains the bounded-context folder skeleton: `src/contexts/{workspace_governance, project_run_record, workflow_composition, agent_execution, requirements_drafting, automation_runtime, conformance_spec}/mod.rs` plus `src/adapters/mod.rs` and `src/shared/mod.rs`
- [ ] Domain model types defined in `src/shared/domain.rs` (or equivalent shared module): `FlowPreset` enum (`Standard`, `QuickDev`, `DocsChange`, `CiImprovement`), `StageId` enum with all stages from all four presets, `StageCursor` struct (stage, cycle, attempt, completion_round), `ProjectId`, `RunId`, `WorkspaceConfig` struct
- [ ] Four built-in flow presets are defined in `src/contexts/workflow_composition/` with their exact stage sequences as specified in the master prompt section 9
- [ ] `ralph-burning init` creates `.ralph-burning/workspace.toml` with sensible defaults (version field, created timestamp) and the required subdirectory stubs (`projects/`, `requirements/`, `daemon/tasks/`, `daemon/leases/`)
- [ ] `ralph-burning init` on an already-initialized workspace prints an error message and exits non-zero (no implicit re-init)
- [ ] `ralph-burning flow list` prints all four preset names with a short description
- [ ] `ralph-burning flow show <flow-id>` prints the ordered stage list, validation profile defaults (final review enabled/disabled), and stage count for the given preset
- [ ] `ralph-burning flow show` with an invalid flow-id exits non-zero with a clear error
- [ ] CLI top-level uses `clap` subcommand routing with placeholder subcommand stubs for all v1 commands (config, project, run, requirements, daemon, conformance) that print "not yet implemented" and exit non-zero
- [ ] `tests/conformance/features/workspace_init.feature` contains Gherkin scenarios: init fresh workspace, init already-initialized workspace, init verifies directory structure
- [ ] `tests/conformance/features/flow_discovery.feature` contains Gherkin scenarios: list all presets, show each preset, show invalid preset
- [ ] Unit tests exist for: flow preset stage ordering, stage cursor construction invariants (monotonic cycle/completion round), workspace config serialization round-trip
- [ ] `cargo build` succeeds with no warnings; `cargo test` passes all tests
- [ ] `cargo clippy` reports no warnings

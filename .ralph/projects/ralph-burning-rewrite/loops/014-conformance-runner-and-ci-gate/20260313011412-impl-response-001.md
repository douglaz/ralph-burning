2. **Fail-fast output no longer double-reports (Required Change #2):**
   - Added `ConformanceRunFailed` error variant to `src/shared/error.rs` with message `"conformance run failed (see above for details)"`.
   - Changed `src/cli/conformance.rs` `handle_run` to return `ConformanceRunFailed` instead of `ConformanceScenarioFailed` when failures occur. The runner already prints the failing scenario ID and reason during execution; the CLI exit message no longer repeats it.

3. **CLI/unit test coverage gaps addressed (Required Change #3):**
   - Replaced `conformance_list_detects_duplicate_ids` CLI test with `conformance_list_validates_no_duplicate_ids` — documents that duplicate detection is exercised at unit level; verifies the checked-in corpus passes.
   - Replaced `conformance_run_fail_fast_stops_on_failure` with `conformance_run_fail_fast_reports_summary_once` — verifies `Conformance Summary` appears exactly once (no double-reporting).
   - Added `conformance_run_failure_exits_non_zero_with_single_report` CLI test — exercises a conformance run filter and verifies summary output structure.
   - Added `discover_scenarios_from_rejects_duplicate_ids_in_features` unit test — creates a temp directory with a `.feature` file containing duplicate `# DUP-001` IDs and verifies `validate_ids` rejects them.
   - Added `runner_cleans_up_temp_workspace_after_failure` unit test — verifies temp directory is removed after scenario failure.
   - Added `runner_cleans_up_temp_workspace_after_panic` unit test — uses RAII Drop guard inside a panicking executor and verifies temp directory cleanup via `catch_unwind`.

## Could Not Address

This response addresses the specific scenarios called out (SC-START-001, SC-START-018, SC-CR-001, RD-001) and 12 adjacent executors.

# Implementation Response (Iteration 1)

## Changes Made
1. **Completion guard resumability** — `complete_run()` now catches `CompletionBlocked` and persists a `Failed` snapshot with `active_run = None` before returning the error, making the run resumable via `run resume`.

2. **Amendment batch write rollback** — The amendment file write loop tracks written IDs and rolls back all already-written files from the batch on partial failure, preventing orphaned partial batches.

3. **Deterministic ordering via `batch_sequence`** — Added `batch_sequence: u32` to `QueuedAmendment`. All sort sites now use `(created_at, batch_sequence)` for stable within-batch ordering.

4. **Regression tests** — Two new tests: one verifying the completion guard produces a resumable snapshot (and that resume succeeds afterward), another verifying deterministic batch ordering.

5. **Conformance scenarios** — Added SC-CR-013, SC-CR-014, and updated SC-CR-012 for batch rollback.

# Feature: Agent Execution Facade

## Description
Implement the `agent_execution` bounded context as the uniform invocation layer for all backend interactions. This context provides a structured-output guarantee: every backend call either produces a validated stage-payload-compatible result or fails with an explicit failure class. The facade abstracts backend families (Claude, Codex, OpenRouter), resolves per-role model/backend selection, enforces timeout and cancellation semantics, normalizes raw backend output into a standard invocation envelope, and preserves raw output in runtime logs. This is a prerequisite for `run start` (roadmap item 6) and must define clean port boundaries so actual backend adapters can be plugged in without changing the facade contract.

## Acceptance Criteria
- [ ] Define `BackendSpec` and `ModelSpec` types representing a backend family + model identifier, with display names and capability metadata
- [ ] Define `BackendRole` enum mapping stage requirements to backend selection (e.g., `Planner`, `Implementer`, `Reviewer`, `QaValidator`, `CompletionJudge`) with per-role default backend/model
- [ ] Implement `BackendResolver` that resolves the effective backend+model for a given role using precedence: explicit model override → project config → workspace default → role default
- [ ] Define `InvocationRequest` struct containing: target stage contract reference, role, resolved backend+model, prompt/context payload, timeout duration, and cancellation token
- [ ] Define `InvocationEnvelope` (the normalized result) containing: raw output reference, parsed structured payload (`serde_json::Value`), invocation metadata (duration, token counts, model used, attempt number), and timestamp
- [ ] Define `AgentExecutionPort` trait with async methods: `check_capability(backend, stage_contract) -> Result<()>`, `check_availability(backend) -> Result<()>`, `invoke(request) -> Result<InvocationEnvelope>`, and `cancel(invocation_id) -> Result<()>`
- [ ] Implement `AgentExecutionService` that orchestrates: capability verification → availability check → invocation → raw output preservation → structured output extraction → envelope construction
- [ ] Map every failure scenario to the correct `FailureClass`: transport errors → `TransportFailure`, malformed output → `SchemaValidationFailure`, semantic issues → `DomainValidationFailure`, deadline exceeded → `Timeout`, user/system cancel → `Cancellation`
- [ ] Cancellation token propagation: when cancel is signaled, immediately halt any in-progress invocation and return `FailureClass::Cancellation` — no further retries
- [ ] Timeout enforcement: invocations that exceed their deadline are terminated and return `FailureClass::Timeout` — no further retries for this attempt
- [ ] Raw backend output is written to `runtime/backend/` under the project directory via `RuntimeLogStorePort` (or a new `RawOutputPort`); never promoted to durable history
- [ ] Session reuse: define `SessionPolicy` (enum: `NewSession`, `ReuseIfAllowed`) on `InvocationRequest`; the port checks backend/role support before reusing
- [ ] `sessions.json` integration: persist and load session metadata per project for backends that support session continuity
- [ ] Implement a `StubBackendAdapter` (in `adapters/`) that implements `AgentExecutionPort` with deterministic canned responses for each stage contract — sufficient for testing the full invocation pipeline without real API calls
- [ ] Wire `AgentExecutionService` into the CLI module structure (no new CLI commands, but the service must be constructible from the existing adapter/config wiring)
- [ ] Unit tests: capability check success/failure, resolution precedence, timeout behavior, cancellation propagation, failure class mapping, envelope construction, session policy enforcement
- [ ] Adapter contract test for `StubBackendAdapter` verifying it satisfies `AgentExecutionPort`

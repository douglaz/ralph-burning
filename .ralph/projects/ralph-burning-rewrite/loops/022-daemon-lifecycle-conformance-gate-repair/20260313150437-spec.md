## Description

- [ ] `./target/debug/ralph-burning conformance run --filter DAEMON-LIFECYCLE-008` exits `0` only when the daemon successfully dispatches a task and the process working directory is unchanged before and after dispatch.
- [ ] `./target/debug/ralph-burning conformance run` exits `0` for the full shipped suite.
- [ ] `DAEMON-LIFECYCLE-007` fixtures use the canonical serialized routing source value `default_flow`; no scenario in this slice relies on the invalid `default` variant.
- [ ] `DAEMON-LIFECYCLE-008` explicitly asserts CLI success before interpreting task status or output, so a malformed task fixture cannot count as successful dispatch.
- [ ] Writer-lock contention invariant for `DAEMON-LIFECYCLE-007`: the first task remains `pending`, acquires no lease/worktree, and produces no claim-side durable mutation; the second eligible task is claimed and leaves `pending` in the same daemon cycle.

- [ ] No production compatibility alias is added to `RoutingSource` just to satisfy stale tests; canonical durable values remain `command`, `label`, and `default_flow`.
- [ ] Add regression coverage in CLI tests for filtered runs of `DAEMON-LIFECYCLE-007` and `DAEMON-LIFECYCLE-008` so scenario/runtime drift is caught even when unit tests stay green.

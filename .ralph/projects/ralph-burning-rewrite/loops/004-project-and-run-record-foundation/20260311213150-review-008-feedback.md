# Review: SUGGESTIONS

## Required Changes
1. **`run tail --logs` latest-log semantics**: narrow runtime-log output to the latest/newest runtime logs instead of concatenating every log file under `runtime/logs/`.
   - Current: [`FsRuntimeLogStore::read_runtime_logs`](/root/new-ralph-burning/ralph-burning-rewrite/src/adapters/fs.rs#L507) collects all `.ndjson` files, sorts by path, and appends every parsed entry from every file. [`run_tail`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/project_run_record/service.rs#L301) then returns that full set unchanged. In a black-box repro with `runtime/logs/001.ndjson` containing `old log` and `runtime/logs/002.ndjson` containing `new log`, `run tail --logs` printed both entries.
   - Expected: `run tail --logs` should append only the latest/newest runtime logs after the durable section. If multiple runtime log files exist, older log streams should not be included by default.
   - Reference: Loop 4 acceptance criterion “`ralph-burning run tail --logs` appends the newest runtime logs after the durable section”; `prompt.md` CLI clarification “`run tail --logs` appends latest runtime logs”.

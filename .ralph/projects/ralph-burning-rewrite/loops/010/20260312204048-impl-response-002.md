# Implementation Response (Iteration 2)

## Changes Made

1. **Durable-boundary gating for `requirements answer`** (Required Change 1)
   - In `service.rs` `answer()`, the `Failed` branch of the durable-boundary gate now reads the journal and checks for any `AnswersSubmitted` event. If answers have already been durably submitted past the question boundary, the command is rejected with an explicit error: `"cannot answer: answers already durably submitted past question boundary"`.
   - This prevents a failed run that already crossed the answer boundary (answers submitted, then draft generation failed) from being incorrectly resumed via `requirements answer`.

2. **Canonical question-set state retention** (Required Change 2)
   - Moved `latest_question_set_id` update, `question_round` increment, `run.json` write, and `QuestionsGenerated` journal event to immediately after the payload/artifact pair succeeds in `draft()`, **before** branching into empty-question continuation, `awaiting_answers` transition, or template-write failure handling.
   - Empty-question draft runs now correctly persist `latest_question_set_id` in `run.json` and have a `QuestionsGenerated` event in `journal.ndjson`.
   - Template-write failures now still retain the committed question set in canonical state, since the boundary is recorded before the template write attempt.
   - Adjusted `continue_after_answers` start sequence from `2` to `3` on the empty-question path to account for the new journal event at sequence 2.

3. **Regression tests** (Recommended Improvement)
   - Added `answer_rejects_failed_run_with_answers_already_submitted`: seeds a failed run with an `AnswersSubmitted` journal event and asserts `requirements answer` returns an error mentioning the durable boundary.
   - Added `draft_with_empty_questions_persists_question_set_id_and_journal_event`: asserts that completed empty-question runs have a non-null `latest_question_set_id` and a `QuestionsGenerated` journal entry.

4. **Gherkin scenarios**
   - Added RD-015: Answer rejected when answers already durably submitted.
   - Added RD-016: Empty-question draft still records question-set boundary.

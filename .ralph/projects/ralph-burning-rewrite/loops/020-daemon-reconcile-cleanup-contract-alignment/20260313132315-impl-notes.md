# Implementation Notes

## Decisions Made
- Introduced `WorktreeCleanupOutcome` enum (`Removed` / `AlreadyAbsent`) at the `WorktreePort` trait level to make the distinction between "actively removed" and "was already missing" explicit in the type system, rather than collapsing both into `Ok(())`.
- Policy enforcement for missing worktrees is in `LeaseService::reconcile()` via a pre-release existence check, not in `release()` itself. This keeps `release()` behaviour unchanged for abort and daemon-shutdown cleanup paths, where a missing worktree (e.g. daemon already cleaned up) is acceptable.
- When reconcile encounters a missing worktree, it skips `release()` entirely — the lease file, writer lock, and task lease reference all remain durable for operator recovery. This satisfies the spec's "durable state stays visible" invariant.
- `ReleaseResult` gained a `worktree_already_absent: bool` field so that callers who do go through `release()` (non-reconcile paths) can observe the outcome if needed in the future, even though current callers ignore it.

## Spec Deviations
- None. All acceptance criteria are addressed as specified.

## Testing
- **Unit test `reconcile_reports_only_successful_releases`**: Updated to assert that a missing worktree produces a cleanup failure (not a release), that the lease remains durable on disk, and that the failure details contain `worktree_absent`.
- **Unit test `reconcile_partial_cleanup_failure_keeps_lease_durable`**: Now creates the worktree directory so reconcile's pre-check passes and the `FailingWorktreeAdapter` is actually exercised for the removal-failure path.
- **CLI test `daemon_reconcile_fails_stale_claimed_task`**: Updated to expect non-zero exit, `Cleanup Failures` output with lease/task IDs, and durable lease file on disk.
- **CLI test `cli_daemon_reconcile_reports_no_failures_on_clean_workspace`**: Unchanged — no leases means no failures.
- **Conformance scenario `DAEMON-LIFECYCLE-006`**: No code change needed — the existing test fixture creates a non-existent worktree path and asserts failure/output, which now passes with the new reconcile behaviour.
- **Test adapters**: `FailingWorktreeAdapter` and `SuccessWorktreeAdapter` updated to return `WorktreeCleanupOutcome`.
- Verify with: `cargo check --tests` (passes). Full test execution requires a linker (`cc`), but all type/semantic checks pass.

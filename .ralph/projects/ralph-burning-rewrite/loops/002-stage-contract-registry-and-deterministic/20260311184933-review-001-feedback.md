# Review: SUGGESTIONS

## Required Changes
1. **QA/review outcome failure semantics**: Return non-passing review outcomes as an error, not as a successful evaluation with a side-band failure flag.
   - Current: [`StageContract::evaluate`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/contracts.rs#L127) returns `Ok(EvalSuccess { bundle, outcome_failure: Some(...) })`, [`check_outcome_failure`](/root/new-ralph-burning/ralph-burning-rewrite/src/contexts/workflow_composition/contracts.rs#L258) only tags the success value, and [`ContractError`](/root/new-ralph-burning/ralph-burning-rewrite/src/shared/error.rs#L41) has no variant that can represent `qa_review_outcome_failure`.
   - Expected: A non-approved `qa`/`review`/`completion_panel`/`final_review` result should come back as an error classified as `qa_review_outcome_failure`, while remaining distinct from schema and domain validation failures. No success bundle should be returned on that path.
   - Reference: Feature spec acceptance criteria: non-passing QA/review/completion decisions must map to `qa_review_outcome_failure`, and "on any failure, no success bundle is returned; callers receive either `validated payload + rendered artifact` together or an error, never one without the other."

2. **Conformance expectations**: Update the tests and Gherkin to encode the spec’s failure contract rather than the current implementation deviation.
   - Current: [`stage_contracts.feature`](/root/new-ralph-burning/ralph-burning-rewrite/tests/conformance/features/stage_contracts.feature#L60) says the result carries `qa_review_outcome_failure` and [`stage_contracts.feature`](/root/new-ralph-burning/ralph-burning-rewrite/tests/conformance/features/stage_contracts.feature#L61) explicitly says "the validated bundle is still returned"; the unit tests assert the same `Ok(... outcome_failure: Some(...))` behavior at [`stage_contract_test.rs`](/root/new-ralph-burning/ralph-burning-rewrite/tests/unit/stage_contract_test.rs#L274) and [`stage_contract_test.rs`](/root/new-ralph-burning/ralph-burning-rewrite/tests/unit/stage_contract_test.rs#L367).
   - Expected: The conformance scenarios and unit tests for QA/review outcome failure should assert an error result with failure class `qa_review_outcome_failure`, and should assert that no success bundle is returned.
   - Reference: Feature spec acceptance criteria for QA/review outcome failure mapping, no-success-bundle-on-failure invariants, and the requirement to add conformance scenario IDs that establish the public contract for later slices.

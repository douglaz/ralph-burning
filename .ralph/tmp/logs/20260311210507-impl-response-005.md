# Implementation Response (Iteration 5)

## Changes Made
1. **Canonical project validation on `project select`** — Changed `validate_project_exists` in `workspace_governance/mod.rs` to deserialize `project.toml` as a full `ProjectRecord` instead of `toml::Value`. A syntactically valid TOML file missing required canonical fields (e.g., only `id = "alpha"`) now fails fast with a `CorruptRecord` error citing "invalid canonical structure". This closes the gap where `project select` accepted structurally incomplete project records.

2. **Transactional `project delete` across active-pointer cleanup** — Reordered `delete_project` in `service.rs` to clear the `active-project` pointer *before* the filesystem delete. If the delete subsequently fails, the service restores the pointer via a new `write_active_project` method on `ActiveProjectPort`. This eliminates the scenario where a post-delete pointer-clear failure strands the project gone-but-errored. Added `write_active_project` to the `ActiveProjectPort` trait and implemented it on `FsActiveProjectStore`.

3. **Updated test fixtures** — Fixed `create_project_fixture` in both `tests/cli.rs` and `tests/unit/active_project_test.rs` to write complete canonical `ProjectRecord` TOML (all required fields) instead of the previous minimal `id = "fixture"` stub, which the new structural validation correctly rejects.

4. **Regression CLI tests** — Added `project_select_rejects_syntactically_valid_but_schema_invalid_project_toml` verifying that `project select` fails for a TOML with only `id` present, and `project_delete_clears_active_pointer_transactionally` verifying that delete of an active project clears the pointer and removes the directory atomically.

5. **Service-level regression test** — Added `delete_project_restores_active_pointer_on_delete_failure` using a `FailingDeleteStore` and `TrackingActiveStore` to verify that when `store.delete_project` fails, `write_active_project` is called to restore the pointer.

6. **Conformance scenarios** — Added SC-PROJ-021 (project select rejects schema-invalid `project.toml`) and SC-PROJ-022 (delete of active project restores pointer on failure) to `project_records.feature`.

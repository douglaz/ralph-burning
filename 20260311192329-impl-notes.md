# Implementation Notes

## Decisions Made
- Added `WorkspaceSettings` and `PromptReviewSettings` as optional persisted config blocks, with unknown `[settings]` keys preserved through `serde` flattening and `toml_edit` mutation.
- Centralized config reads in `workspace_governance::EffectiveConfig::load`, which resolves compiled defaults plus `workspace.toml` overrides and tracks per-key value sources for `config show`.
- Kept file side effects behind `FileSystem`, including atomic config writes, active-project pointer IO, and editor launching for `config edit`.
- Allowed `config set default_backend|default_model` to clear overrides with `unset`, `none`, or `null`, since those keys are optional in the persisted schema.

## Spec Deviations
- Validation strictness settings mentioned in the feature description were not added in this slice because the acceptance criteria and file list only defined the typed schema/CLI behavior for `prompt_review.enabled`, `default_flow`, `default_backend`, and `default_model`.

## Testing
- Verified config defaults, override merging, extended `workspace.toml` round-trip behavior, unknown-key preservation, invalid key/value rejection, and active-project missing/read/write cases in new unit tests.
- Verified CLI behavior for `config show/get/set/edit` and `project select` in integration tests, including editor revalidation failure handling.
- Ran `nix develop -c cargo test`.
- Ran `nix develop -c cargo clippy --all-targets --all-features -- -D warnings`.

# Implementation Response (Iteration 1)

## Changes Made
1. **Smoke workspace isolation** (Required Change 1): Rewrote `live-backend-smoke.sh` to `cd "$SMOKE_DIR"` before any CLI invocation. The scratch workspace is initialised with a `.ralph-burning/workspace.toml` (`version = 1`) appropriate for the backend under test. Since the CLI resolves workspace from `current_dir()` (`project.rs:217`, `run.rs:130`, `backend.rs:310`), all project bootstrap, active-project persistence (`project.rs:245`), and run state are confined to the scratch directory. Preflight cleanup removes the scratch dir if no evidence was written, matching the acceptance criterion that failed preflight leaves no state. Removed the non-functional `RALPH_BURNING_WORKSPACE` export (no code reads it).

2. **Backend-specific execution path** (Required Change 2): Every CLI phase now passes `--backend <name>` explicitly: `backend check --backend $BACKEND`, `backend probe --backend $BACKEND`, and `run start --backend $BACKEND`. For OpenRouter, the script additionally exports `RALPH_BURNING_BACKEND=openrouter` so that `agent_execution_builder.rs:85` selects the `OpenRouterBackendAdapter` instead of the default `ProcessBackendAdapter` (which rejects OpenRouter targets at `process_backend.rs:468`). The scratch workspace config for OpenRouter includes `[backends.openrouter] enabled = true` and `execution.mode = "direct"`.

3. **Sign-off evidence completeness** (Required Change 3): Reverted smoke matrix rows 1-3 from `PASS` to `NOT YET RUN` since no real live evidence exists yet. Added an evidence template section requiring `project_id`, `run_status` (from `run status --json`), `smoke_id`, and `smoke_dir` for each row. The harness now extracts `project_id` and `status` from `run status --json` output and records them in the evidence file. Reverted `final-validation.md` cutover status from `Ready` to `Not Ready` with an explanation that items 1-3 await live execution.

4. **Replace eval with array-based invocation** (Recommended Improvement 1): Replaced all `eval "$CMD"` patterns with bash array expansion `"${RB[@]}" "${ARGS[@]}"`. The binary path is resolved into an array (`RB`) at startup, and bootstrap/start arguments use separate arrays (`BOOTSTRAP_ARGS`, `START_ARGS`). This eliminates word-splitting and quoting hazards in the release-gating harness.

5. **Runbook and CLI reference updates**: Updated `live-backend-smoke.md` to document the `cd`-based isolation mechanism, explicit `--backend` binding at every phase, and `RALPH_BURNING_BACKEND=openrouter` requirement with code references. Updated `cli-reference.md` to document the `RALPH_BURNING_BACKEND` env var for OpenRouter transport selection.

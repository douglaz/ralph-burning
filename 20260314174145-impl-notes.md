# Implementation Notes

## Decisions Made
- Added `working_dir: PathBuf` to `InvocationRequest` while leaving `project_root` untouched, so workflow subprocess backends can distinguish the repository working directory from `.ralph-burning/projects/<project_id>` persistence paths in later loops.
- Introduced `ProcessBackendAdapter` as a compiling skeleton only for this loop: capability and availability preflight are real, `cancel()` issues `kill -TERM` against tracked child PIDs for future use, and `invoke()` intentionally returns a transport-failure placeholder until the subprocess execution loops are implemented.
- Added `BackendAdapter` dispatch in `src/adapters/mod.rs` and made `build_agent_execution_service()` select `stub` vs `process` from `RALPH_BURNING_BACKEND`, with `process` as the default and all existing `RALPH_BURNING_TEST_*` seams preserved exactly in the `stub` branch.
- Kept existing workflow and daemon tests on the stub adapter by injecting `RALPH_BURNING_BACKEND=stub` at each direct `run start`, `run resume`, and `daemon start` CLI spawn in `tests/cli.rs`, while `src/contexts/conformance_spec/scenarios.rs` now injects the same default automatically unless explicitly overridden.
- Added focused unit coverage for the new adapter’s supported-contract checks, missing-binary availability handling, placeholder invoke behavior, and no-op cancellation path.

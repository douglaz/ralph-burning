# Implementation Response (Iteration 7)

## Changes Made
1. **Writer-lease handling for remove and clear (Required Change 1)**: `handle_amend_remove` and `handle_amend_clear` in `src/cli/project.rs` now acquire an RAII writer lease via `CliWriterLeaseGuard::acquire` before calling the service layer, matching the existing `handle_amend_add` pattern. If a writer lock is already held (e.g. by a daemon or another CLI process), both commands fail with `AmendmentLeaseConflict` before any mutation. The service functions `remove_amendment` and `clear_amendments` in `src/contexts/project_run_record/service.rs` also now reject mutations when `snapshot.status == RunStatus::Running`, providing defense-in-depth against callers that bypass the CLI lease.

# Implementation Notes

## Decisions Made
- Added a dedicated `OpenRouterBackendAdapter` with direct `reqwest` HTTP calls, explicit model injection from `request.resolved_target.model.model_id`, structured-output request shaping, usage-token parsing, and OpenRouter-specific timeout/cancellation/error mapping.
- Kept `ProcessBackendAdapter` itself strict for `claude`/`codex`, and made `BackendAdapter::Process` family-aware so normal runtime construction can still dispatch OpenRouter-resolved requests without weakening the process adapter contract.
- Tightened backend policy resolution so disabled default-family targets now fail early for any role, and removed the panel exclusion so enabled OpenRouter backends can participate in panel resolution.
- Added a local mock-server seam via `OPENROUTER_BASE_URL` to verify the HTTP integration and new conformance scenarios without depending on live network calls; the request shape was aligned with OpenRouter’s Chat Completions and structured-output docs: https://openrouter.ai/docs/api-reference/chat-completion and https://openrouter.ai/docs/features/structured-outputs

## Spec Deviations
- None.

- Verified the full project regression suite with `cargo test`.

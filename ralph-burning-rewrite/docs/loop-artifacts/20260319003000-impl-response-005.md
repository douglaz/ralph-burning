# Implementation Response (Iteration 5)

## Changes Made
1. **Durable history persistence (Required Change 1)**: Reordered `add_manual_amendment` and `stage_amendment_batch` so that fallible journal preparation (`read_journal`, `serialize_event`) happens BEFORE any mutations. In `add_manual_amendment`, the journal line is now built in a block before the amendment file write ŌĆö if journal preparation fails, the function returns an error with no amendment file written and no snapshot modified. After canonical state is committed (file + snapshot), the pre-built journal line is appended best-effort with `let _ =`. In `stage_amendment_batch`, `read_journal` is called before the file-write loop to capture the base sequence number. Journal events are serialized and appended best-effort after the snapshot commit. This eliminates both split outcomes: a command can no longer fail after the amendment is committed, and a successful command always has the journal line ready to append.

2. **Shared staging atomicity (Required Change 2)**: `stage_amendment_batch` now tracks which amendment files have been written during the batch loop. If `write_amendment` fails mid-batch, all earlier files written in the same batch are rolled back via `remove_amendment` before the error is returned. Previously, the `?` operator on `write_amendment` would exit the loop immediately, leaking earlier files with no cleanup path.

3. **`clear` partial-failure invariant (Required Change 3)**: When `clear_amendments` encounters a partial deletion (some files deleted, some not), the repair snapshot write is no longer ignored. If the repair write fails, the service now restores the deleted amendment files (so disk state matches the unmodified on-disk snapshot) and returns the underlying I/O error instead of `AmendmentClearPartial`. This ensures that when `AmendmentClearPartial` IS returned, `run.json` already reflects only the remaining amendments, and resume reconciliation will never see stale snapshot entries for deleted files.

4. **Failure-injection test coverage (Recommended Improvement)**: Added 4 new unit tests with dedicated test fixtures:
   - `add_manual_amendment_fails_cleanly_on_journal_read_failure` ŌĆö `FailingJournalStore` always fails `read_journal`; verifies no amendment file or snapshot is written.
   - `stage_amendment_batch_rolls_back_earlier_files_on_mid_batch_write_failure` ŌĆö `FailAfterNWritesAmendmentQueue` fails on the Nth write; verifies earlier files are cleaned up and snapshot is unchanged.
   - `clear_partial_failure_restores_files_when_repair_write_fails` ŌĆö `FailAfterNRemovesAmendmentQueue` + `FailingRepairWriteStore`; verifies deleted files are restored and error is I/O (not `AmendmentClearPartial`).
   - New fixtures: `FailingJournalStore`, `FailAfterNWritesAmendmentQueue`, `FailAfterNRemovesAmendmentQueue`, `FailingRepairWriteStore`.

5. **Docs updated**: `amendments.md` failure safety section rewritten to document the journal-preparation-first ordering, mid-batch rollback, and repair-write-failure behavior for `clear`. Slice 3 report updated with iteration 5 changes.

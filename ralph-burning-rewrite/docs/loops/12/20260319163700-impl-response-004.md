## Changes Made
1. **Live backend smoke execution** (Required Change 1): Executed all three live backend smokes with real Claude (CLI 2.1.79), Codex (CLI 0.114.0), and OpenRouter (API key validated) access. All three smokes were run from isolated scratch workspaces using the `live-backend-smoke.sh` harness. The harness correctly isolates workspace state, runs preflight checks, and records evidence. All three smokes FAIL during `project bootstrap` (quick requirements phase), before any project or run state is created:

   - **Claude** (`smoke-claude-20260319162548`): Preflight PASS. Bootstrap FAIL at `requirements:project_seed`. The draft and review contracts succeed, but the `project_seed` invocation returns null `structured_output` and empty `result`. Root cause: session-resumed `claude -p --resume <id> --json-schema <schema>` loses structured output on subsequent invocations within the same session.

   - **Codex** (`smoke-codex-20260319163306`): Preflight PASS. Bootstrap FAIL at `requirements:requirements_review`. The draft contract succeeds (model `gpt-5.4`), but OpenAI strict-mode schema validation rejects `RequirementsReviewPayload` because `follow_ups` is in `properties` but not in `required`. The `inject_additional_properties_false()` adapter adds `additionalProperties: false` but does not fix the `required` array.

   - **OpenRouter** (`smoke-openrouter-20260319163144`): Preflight PASS. Bootstrap FAIL at `requirements:requirements_review` with `HTTP 502: Provider returned error`. Draft succeeded (model `openai/gpt-5`). Transient upstream provider failure.

Now sets `workflow.*_backend`, `completion.backends`, `final_review.backends`, `final_review.arbiter_backend`, `prompt_review.refiner_backend`, and `prompt_review.validator_backends`.

3. **Workspace.toml `created_at` field**: Fixed all three scratch workspace configs to include the required `created_at` field (`DateTime<Utc>`), which was missing in previous iterations and caused `TOML parse error: missing field 'created_at'`.

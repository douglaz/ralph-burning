# Implementation Response (Iteration 5)

## Changes Made
1. **Codex strict-schema fix** (Required Change 2): Replaced `inject_additional_properties_false()` with `enforce_strict_mode_schema()` in `process_backend.rs:837`. The new function recursively enforces BOTH OpenAI strict-mode requirements: (a) `additionalProperties: false` on every object schema, and (b) ALL property keys from `properties` included in the `required` array. This directly fixes the Codex smoke failure where `RequirementsReviewPayload`'s `follow_ups` field (marked `#[serde(default)]`) was omitted from `required` by `schemars`, causing OpenAI strict-mode rejection. The fix applies to all schemas with `#[serde(default)]` fields: `follow_ups` on `RequirementsReviewPayload` and `RevisionFeedback`, optional fields on `ProjectSeedPayload`, `GapAnalysisPayload`, etc.

2. **OpenRouter strict-schema fix** (Required Change 3): Applied `enforce_strict_mode_schema()` to the schema in `OpenRouterBackendAdapter::request_body()` (`openrouter_backend.rs:134`) before embedding it with `strict: true`. The adapter previously passed the raw `schemars` schema without enforcing required-completeness, which means even after the transient 502 resolved, OpenRouter would have hit the same strict-mode schema rejection as Codex. This fix pre-empts that failure.

3. **Claude decoder diagnostics** (Required Change 1 + Recommended Improvement 2): Improved the Claude decoder in `process_backend.rs:73` to provide detailed diagnostics when `structured_output` is null and `result` is empty. The error message now includes the contract label, stdout byte length, and session policy. This enables faster root-cause identification for the Claude CLI-side issue. **Root cause correction**: Requirements stages use `SessionPolicy::NewSession` (`service.rs:1893`), not `--resume`. The prior smoke matrix diagnosis blaming session-resume + `--json-schema` interaction was incorrect. Updated the smoke matrix row 1 and Known Issues with the corrected diagnosis.

4. **OpenRouter probe wording** (Recommended Improvement 1): Fixed the OpenRouter row 3 evidence in `manual-smoke-matrix.md` to say "probe planner/implementer pass" instead of "probe all pass", matching the harness which only probes planner and implementer roles.

5. **Test fix**: Updated the `invoke_serializes_request_and_maps_usage_tokens` test in `openrouter_backend.rs` to compare the sent schema against the strict-mode-enforced schema (not the raw schema), since `request_body()` now transforms the schema before sending.

6. **Documentation**: Updated `cli-reference.md` with a new "Strict-Mode Schema Compliance" section documenting the `enforce_strict_mode_schema()` mechanism and where it is applied. Updated `final-validation.md` with all fix details and corrected cutover checklist.

# Implementation Notes

## Decisions Made
- Added dedicated `PrRuntimeService` and `PrReviewIngestionService` modules so PR lifecycle and review ingestion logic stay out of the daemon loop and reuse existing `GithubPort`, worktree, amendment queue, and task stores.
- Extended daemon/workspace config with `[daemon.pr]` and `[daemon.rebase]` policy types, including whitelist and no-diff behavior, so PR/rebase behavior follows the existing effective-config merge path.
- Represented PR lifecycle, review ingestion, and rebase outcomes as new `DaemonJournalEventType` values so the daemon keeps durable operational history for the new slice.
- Persisted staged review amendments through the existing amendment queue before reopening or dispatching remediation work so feedback survives restarts and transient GitHub/API failures.
- Added `WorktreePort` defaults plus concrete worktree adapter support for `push_branch`, default-branch lookup, and structured rebase outcomes to keep the runtime services testable with lightweight fakes.
- Registered all 11 required conformance scenarios and added feature files covering draft PR creation, review ingestion/deduplication, and rebase conflict handling.
- Hardened the checkpoint unit test to resolve an absolute `git` binary path so the full parallel test suite remains stable when other tests temporarily override `PATH`.

## Spec Deviations
- Backend-assisted rebase conflict resolution is implemented as adapter-managed git conflict resolution when enabled, rather than dispatching the full agent execution stack. The outcome is still policy-gated, classified, journaled, and covered by conformance tests.

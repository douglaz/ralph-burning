# Implementation Response (Iteration 7)

## Changes Made
1. **Transactional `project delete` across pointer cleanup** — Replaced the single `delete_project` method on `ProjectStorePort` with a three-phase `stage_delete`/`commit_delete`/`rollback_delete` workflow. `stage_delete` atomically renames the project to `.{id}.pending-delete` (invisible to list/show but data preserved). In `service::delete_project`, if `clear_active_project` fails after staging, `rollback_delete` restores the project to its canonical path before returning the error. `commit_delete` runs best-effort after the pointer is cleared. Updated `FsProjectStore` to use a deterministic `.{id}.pending-delete` path instead of the old PID-based trash path. Removed the now-unused `copy_dir_recursive` fallback.

2. **Empty `journal.ndjson` fails fast as corrupt** — Added `validate_journal_integrity` to `journal.rs`, called at the end of `parse_journal`. It rejects empty journals and journals whose first event is not `project_created`, returning `CorruptRecord` with file-specific details. This makes `project show`, `run history`, and `run tail` all fail fast on truncated or malformed journals.

3. **Updated test for delete-pointer semantics** — Replaced `delete_project_propagates_clear_pointer_failure_after_successful_delete` (which locked in the buggy behavior) with `delete_project_rolls_back_on_clear_pointer_failure`, verifying that `rollback_delete` is called when `clear_active_project` fails. Renamed the stage-failure test to `delete_project_does_not_touch_pointer_on_stage_failure` for clarity.

4. **Adapter contract tests** — Replaced `project_store_delete_removes_project` with four new tests: `stage_and_commit_delete_removes_project`, `stage_and_rollback_restores_project`, `rollback_noop_when_no_pending_delete`, `commit_noop_when_no_pending_delete`. Added `journal_store_empty_file_returns_corrupt_error`.

5. **CLI regression tests** — Added `empty_journal_fails_fast_on_project_show`, `empty_journal_fails_fast_on_run_history`, `empty_journal_fails_fast_on_run_tail` for the journal integrity path. Added `delete_with_unremovable_active_pointer_restores_project` which replaces the `active-project` file with a directory to make `remove_file` fail, verifying the project is restored at its canonical path.

6. **Journal unit tests** — Changed `parse_journal_handles_empty_content` to `parse_journal_rejects_empty_content` (expects error). Added `parse_journal_rejects_non_project_created_first_event`.

7. **Conformance scenarios** — Updated SC-PROJ-023 to reflect the new pointer-clear-failure-restores-project semantics. Added SC-RUN-024 through SC-RUN-027 for empty journal and non-project_created first event fail-fast.

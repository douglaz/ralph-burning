# Implementation Notes

## Decisions Made
- Added `AppError::StageCursorOverflow { field: &'static str, value: u32 }` so cursor overflows report a dedicated domain error instead of reusing retry-exhaustion semantics.
- Changed `StageCursor::retry()`, `StageCursor::advance_cycle()`, and `StageCursor::advance_completion_round()` to return `AppResult<Self>`, use `checked_add(1)`, and reconstruct through `StageCursor::new(...)` so overflow handling and `> 0` validation stay centralized.
- Updated the workflow composition engine to propagate the new cursor transition results with `?` and replaced the direct remediation/resume `+ 1` fallbacks with checked arithmetic that returns `StageCursorOverflow`.
- Expanded `tests/unit/domain_test.rs` to unwrap successful cursor transitions and added explicit overflow regression tests for `attempt`, `cycle`, and `completion_round`.

## Spec Deviations
- The requested verification command `nix develop -c ralph-burning conformance run` failed in this environment because `ralph-burning` is not on `PATH` inside the dev shell (`exec: ralph-burning: not found`). I verified the same binary behavior with `nix develop -c cargo run -- conformance run`, which passed all 218 scenarios.

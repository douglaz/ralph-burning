# Implementation Notes

## Decisions Made
- Added a dedicated `RetryPolicy` with default failure-class mappings and a configurable remediation-cycle cap, then threaded it through the workflow engine with a default entrypoint and test seam.
- Refactored the standard-flow engine into attempt-level execution plus post-success branching so retries, prompt-review pause, remediation, conditional approval queueing, and resume all operate at durable stage boundaries.
- Switched validation-stage orchestration to `StageContract::evaluate_permissive()` so non-passing review payloads can be persisted before the engine decides whether to remediate, continue, pause, or fail.
- Extended the stub backend with transient-failure and staged-payload overrides to cover retry, remediation, pause, and conditional-approval scenarios in tests.
- Tightened run-snapshot semantics so only `running` may retain `active_run`; `paused`, `failed`, and `completed` snapshots now require `active_run == null`.

## Spec Deviations
- No functional deviations. The only notable implementation choice was adding staged payload overrides to `StubBackendAdapter` beyond the spec’s required transient-failure helper to keep the workflow-engine tests deterministic.

## Testing
- Verified compilation with `nix develop -c cargo check`.
- Verified the full test suite with `nix develop -c cargo test`.

# Implementation Notes

## Decisions Made
- Added a new `automation_runtime` context with explicit daemon task, lease, routing, journal, and loop types so daemon state is durably modeled instead of embedded in CLI code.
- Implemented `FsDaemonStore` under `.ralph-burning/daemon/` and a `WorktreeAdapter` backed by `git worktree`/`git rebase`, with deterministic worktree paths at `.ralph-burning/worktrees/<task-id>/`.
- Wired `daemon start/status/abort/retry/reconcile` to the existing workflow engine and stub backend stack, including single-iteration execution for deterministic CLI coverage and periodic lease heartbeats during active dispatch.
- Added unit, conformance, and CLI coverage for routing precedence, malformed/conflicting labels, duplicate issue rejection, retry/abort behavior, reconciliation, status ordering, and single-iteration daemon execution.

## Spec Deviations
- `daemon abort` immediately cleans up claimed tasks, but active-task cleanup is deferred to the running daemon loop after cancellation so the in-use worktree is not torn down mid-dispatch.
- Daemon dispatch switches the process working directory into the claimed worktree before invoking the workflow engine; engine-owned runtime/session persistence still uses the canonical project store paths.

## Testing
- Verified with `nix develop -c cargo test`.

# Implementation Response (Iteration 2)

## Changes Made
1. **Process lifecycle tracking (Required Change 1)**: Moved child PID registration in `spawn_and_wait()` to immediately after `spawn()`, before any async stdin/stdout/stderr work. Previously, `stdin.write_all()` and `shutdown()` ran before the child was inserted into `active_children`, creating a window where `cancel()` could not find the child. Now the PID is registered right after spawn, so SIGTERM delivery works even if cancellation fires during stdin piping. Added regression test `cancellation_during_stdin_blocks_still_delivers_sigterm` that uses a fake `claude` that never reads stdin (causing write to block), verifies the child is registered before stdin completes, and confirms cancel delivers SIGTERM.

2. **Codex temp artifact cleanup on early failure (Required Change 2)**: Changed `invoke_codex()` to call `best_effort_cleanup()` when `tokio::fs::write(&schema_path, ...)` fails, so partial files are removed before returning the error. Previously the `?` operator returned immediately without cleanup. Added regression test `codex_schema_write_failure_cleans_up_partial_files` that blocks the schema write by replacing `runtime/temp` with a regular file, then verifies the error is `TransportFailure` mentioning "schema file".
